    Akamai ストリームのパッケージングと Telestream Wirecast を使用したライブストリームの作成

    このトピックでは、Akamai ストリームパッケージングと Telestream Wirecast を使用してライブストリーミングイベントをブロードキャストする方法について説明します。

    このトピックでは、Akamai ストリームパッケージング、テレストリーム Wirecast エンコーダー、 リモートアセット動画、および Brightcove Player を使用してライブイベントをブロードキャストするのに役立ちます。他のエンコーダーも使用できます。Brightcove ブのライブストリーミング動画配信オプションの一覧については「ライブストリーミング動画を配信する」を参照してください。

    Akamai ストリームパッケージを使用してライブストリームをブロードキャストするには、以下の操作を行います。

    • ライブストリーミング用に Video Cloud アカウントを有効にする
    • Media モジュールを使用してライブストリームビデオを作成する
    • ライブイベントの Telstream Wirecast を構成する
    • ライブイベントの公開コードを生成する

    Video Cloudアカウントでライブストリーミングを有効にする

    ライブストリーミングイベントを実施する前に、Brightcove ブサポートに連絡して、Video Cloudアカウントでライブストリーミングを有効にしたいことを知らせる必要があります。また、アカウントでのストリームパッケージングの有効化も要求してください。アカウントが設定されると、ライブストリーミングの設定に必要な資格情報と設定情報のセットが記載されたメールが届きます。

    注：Akamai HD用の資格情報はお客様のエンコーダーIPに基づき、地域に応じて提供されます。新しい場所でイベントを開催する予定の場合、新しい場所のエンコーダーのping可能な外部IPアドレスをBrightcove ブサポートに提供する必要があります。そうすることによって、Akamaiはできるだけ近い場所のエントリーポイントを再提供できます。また、公開インターネット上の移動をできるだけ少なくすることによって、待機時間や中断のリスクを小さくできます。変更がAkamaiのネットワーク内部で伝播されるには、ある程度の時間がかかるので、Brightcove ブサポートは前もってこれを十分知っておく必要があります。使用するネットワーク上に外部からpingできるIPアドレスがあれば、エンコーダーがその場所にまだなくても構いません。

    Mediaモジュールを使用してライブストリーム動画を作成する

    ライブストリームを扱うには、自身の動画ライブラリー内に、リモートアセットで動画を作成する必要があります。動画ファイルを自身のシステムやサードパーティ製のシステムで維持管理し、動画の作成時に動画ファイルのURLをBrightcove ブに提供する場合、リモートアセットを使用します。リモートアセットを作成するには、Mediaモジュールまたは CMS API を使用します。

    Mediaモジュールを使用してリモートアセット動画を作成するには、次の手順を行います。

    1. Media モジュールを開きます。
    2. サイド・ナビゲーション・パネルの+リモートビデオの追加をクリックします。
    3. 動画名を入力します。
    4. 動画タイプをライブに設定します。
    5. [ 形式 ] を [ HLS ] に設定します。
    6. レンディション URL については、Brightcove サポートからのメールに記載されている ARL 値から始めます。ARL URL は次の形式になります。 
       http://bcliveunivsecure-lh.akamaihd.net/[EVENT_ANGLE_BITRATE]@684348
      EVENT および ANGLE プレースホルダを値に置き換える必要があります。ビットレートは使用されません。たとえば、URL は次のようになります。 
       http://bcliveunivsecure-lh.akamaihd.net/Live_1@684348

      URL の akamaihd.net の部分の後に、URL に I を挿入します。

       http://bcliveunivsecure-lh.akamaihd.net/i/Live_1@684348

      また、URL の末尾に /master.m3u8 を追加します。最終的なレンディションURLは、以下のようになるはずです。

      http://bcliveunivsecure-lh.akamaihd.net/i/Live_1@684348/master.m3u8
    7. フォーマットされた URL を [URL] フィールドに貼り付け、[レンディションを追加] をクリックします。
    8. [保存]をクリックして、リモートアセット動画を保存します。
    9. リモートアセットがMediaモジュールに表示されるのを確認します。必要に応じてリモートアセット動画をアクティブ化します。

    Mediaモジュールでリモートアセット動画を作成した後、メタデータを他の動画と同様に編集できます。

    ライブイベント用にTelestream Wirecastを設定する

    このセクションでは、ライブストリーミングイベントをサポートするようにTelestream Wirecastエンコーダーを設定し、使用します。Telestream Wirecastソフトウェアは、カメラから入力を取り込み、CDNによって配信できるストリームを生成するデスクトップアプリケーションです。ハードウェアベースおよび他のソフトウェアベースのエンコーディングソリューションには、ご利用のライブストリーミングイベントの配信に、より適したものが存在する可能性があります。Wirecastソフトウェアの体験版を TelestreamのWebサイトからダウンロードすることができます。

    注：先へ進む前に、ご利用のコンピューターにWirecastソフトウェアをインストールし、カメラを接続しておく必要があります。

    ライブイベント用にWirecastを設定するには、次の手順を行います。

    1. Wirecastを開きます。
    2. [ + ] をクリックし、[ ビデオキャプチャ ] を選択します。
    3. リストからカメラを選択します。[追加]をクリックします。
    4. 選択したビデオソースがプレビューペインに表示されていることを確認します。
    5. 矢印ボタン ( ) をクリックして、カメラをライブショットにします。
    6. ストリームボタン（）をクリックします。出力設定を入力するよう表示があります。
    7. 配信先を RTMPサーバーに設定します。[OK]をクリックします。
    8. アドレスには、Brightcove ブサポートからのメールに記載されている [ プライマリ Entrypoint ] の値を使用します。
    9. ストリームには、リモートアセット URL を定義するときに使用された [EVENT_ANGLE_BITRATE] @394443 値を使用します。リモートアセットを作成するときに使用した EVENT と ANGLE には、必ず同じ値を使用してください。BITRATE の値を Wirecast で設定したビットレートに設定します。
    10. [資格情報の設定...]をクリックします。
    11. Brightcove ブサポートからの電子メールに記載されたユーザー名とパスワードを入力します。[OK]をクリックします。
    12. [OK]をクリックし、出力設定を保存します。
    13. ストリームボタンをクリックします。ライブストリームが始まるはずです。

    ライブイベント用にパブリッシングコードを生成する

    1. Mediaモジュールを開きます。
    2. ライブストリームのリモートアセット動画を選択します。
    3. [ パブリッシュして埋め込む ] をクリックします。
    4. プレーヤーを選択します。
    5. プレーヤーURLをクリックして、ライブストリームをブラウザーでプレビューします。

    さらにサポートが必要な場合

    ライブイベントを機能させる上でサポートが必要な場合、Brightcove ブサポートをご利用いただけます。こちらから問題を送信することができます。なるべく早くご返信ができるよう、問題解決のためにBrightcove ブサポートチームが必要とする情報を以下にリストアップしています。

    • ストリームに見受けられる特定の兆候。たとえば、一部が再生できない、スムーズに再生できないなど
    • 問題のストリームが過去に正しく機能していたか
    • エンコーダーで使用しているエントリーポイントURL
    • ストリーム名
    • 使用しているエンコーディングソフトウェアおよびハードウェア
    • ライブイベントを公開したプレーヤーのURL
    • ライブリモートアセットの動画ID
    • エンコーダーから公開ポイントホストへのルートトラッキングの結果
