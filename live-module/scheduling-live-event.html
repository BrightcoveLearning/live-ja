--- title: ライブ SEP イベントのスケジューリング parent: Live Module ---

静的エントリポイントを使用したライブイベントの自動開始/停止

この機能を使用すると、1 つまたは複数の Live Job イベントの開始時刻と停止時間をスケジュールおよび管理できます。

前提条件

イベントをスケジュールするには、SEP（静的エントリポイント）タイプのライブイベントを作成する必要があります。

現在、冗長グループのスケジューリングはサポートされていません。

推奨事項とベストプラクティス

最良の結果を得るには、以下の指示に従ってください。

  1. 一度に6つの保留中のアクティベーションをスケジュールできます。さらに必要な場合は、カスタマーサポートにお問い合わせください。
  2. SEP がアクティブ化されると、デフォルトの [最大待機時間] は引き続き有効です（ライブ API を参照）。詳細については、静的エントリポイント )。
  3. 静的エントリポイントで説明されている制限は引き続き有効であることに注意してください。

コントロールルームのスケジューリングパネル

ライブアカウントで自動開始/停止が有効になっている場合、コントロールルームの右上隅にあるプレビュープレーヤーの下に、スケジュールパネルが表示されます。

自動開始/停止を使用するには、以下の手順に従ってください。

  1. ライブモジュールで、静的エントリポイントを使用して新しいライブイベントを作成するか、以前に作成したイベントにアクセスします。
  2. コントロールルームでは、スケジューリングという名前の新しいパネルが利用可能になります。[ +スケジュールの追加] をクリックして、新しい自動アクティベーションと非アクティブ化を作成します。
  3. 開始日と終了日をそれぞれ目的のアクティブ化と非アクティブ化に変更します。
    複数のスケジュールが同時に実行されるのを防ぐことはできませんが、競合する行には警告が表示されます。競合するスケジュールは、アクティベーション時間より前に対処しないと失敗する可能性があります。警告を回避するには、行の 1 つを削除するか、時間を調整するだけです。
  4. 2 番目のスケジュールを作成するには、[ + スケジュールの追加]ボタンをもう一度クリックします。[ + スケジュールの追加 ] ボタンは、保留中のアクティベーションの制限に達すると自動的に無効になります。
  5. スケジュールを削除するには、 アイコン。
  • スケジュールが競合している場合、スケジュールイベントは警告として黄色に変わります。
  • 何らかの理由でストリーミングで障害が発生すると、スケジュールされたイベントが赤に変わります。
  • 失敗したスケジュールは7日間表示されたままになります。

今後のアクティベーションの表示

  1. スケジュールされたアクティベーションを作成したら、[ イベントの管理] > [スタンバイ] タブでそれらを表示できます。次の開始日と終了日は、「近日予定」タイトルの下に表示されます。
  2. [ 表示 ] 選択ボックスでは、スケジュールされたイベントのみを表示できます。
  3. イベントが開始されると、に表示されます住むタブ、および始める時間は無効になります。
