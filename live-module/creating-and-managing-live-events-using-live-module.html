Live モジュールを使用したライブイベントの作成と管理

このトピックでは、Live モジュールを使用してライブイベントを作成および管理する方法を学習します。
 

はじめに

Live モジュールは、ライブイベントをデスクトップとモバイルデバイスの両方にブロードキャストするのに役立ちます。 Live モジュールにイベント情報を入力すると、オンサイトのエンコーダに接続する一連のエンコーダ設定が提供されます。 すべてのトランスコーディングはクラウドで実行されるため、ハードウェアベースのトランスコーダは必要ありません。

Live サブスクリプションによっては、イベント、チャンネル、またはその両方を作成できる場合があります。 通常、イベントは、短い長さ、1 回だけ、または定期的なイベントをストリーミングするために作成されます。 チャンネルは通常、年中無休のライブイベントを配信するときに使用されます。

注:ライブストリーミングのサポートは、Video Cloud Pro および Enterprise のパブリッシャーでのみ利用できます。

前提条件

ライブイベントをブロードキャストするには、次の項目が必要です。

  • Live モジュールにアクセスできる Video Cloud Pro アカウントまたはエンタープライズアカウント
  • イベントをブロードキャストするカメラ
  • Telestream の Wirecast や オープンブロードキャスターソフトウェアなど、独自のエンコーディングソフトウェア が必要です。

制限事項

  • Media モジュールで DVR を使用してライブストリームを作成するには、まずそこから新しいリモートアセットを作成し、URL ストリームの末尾に「playlist_dvr.m3u8」拡張子を付けてライブモジュールのプレイリストを入力する必要があります。
  • 現在、SSAI を使用したライブストリームは Live モジュールで公開できません。 この機能を使用してライブストリームを公開する場合は、Media Module から公開する必要があります。
  • ライブ放送では、20 ～ 30 秒の遅延が発生します。
  • ストリーミングは、イベント開始から 30 分以内に開始する必要があります（デフォルトレベルのアカウント設定）。
  • Internet Explorer を使用するユーザーには、バージョン 10 以降が必要です。
  • DVR を使用したライブ配信は、ブライトコーブプレーヤー SDK の使用時にはサポートされません。 この場合、DVR 再生 URL をリモートアセットに追加し、リモートアセット URL を SDK で使用する必要があります。

ライブイベントの管理

Live モジュールにアクセスするには、Video Cloud Studio にログインし、ナビゲーションヘッダーの [ Live ] をクリックします。 ログイン後、ページの右上隅に Video Cloud アカウント名が表示されます。 複数のアカウントがある場合は、アカウント切り替えのドロップダウンを使用して、使用するアカウントを選択します。 選択したアカウントが Live で有効になっていない場合は、「Brightcove Live」の紹介動画ページが表示されます。 [イベントの管理] ページには、次の 3 つのメインタブが表示されます。

  • ライブ -進行中のライブイベントを一覧表示します。
  • Standby：スタンバイ モードの 繰り返しイベントを 一覧表示します。
  • Completed ：完了したイベントを一覧表示します。
注：ライブチャンネルを作成する機能は、ライブサブスクリプションに基づいています。

[ 表示 ] ドロップダウンを使用して、[ 完了 ] タブに表示されるイベントをフィルタリングできます。

ページの本文には、関連情報を含むライブイベントのリストが表示されます。

  • Job Status：ライブストリームのステータス
    • 処理 -ストリームはライブです
    • 処理エラー -ライブストリームでエラーが発生しました
    • Standby ：ストリーム起動準備完了（静的エントリー・ポイントのみ）
    • Stopp@@ ing -ライブストリームが停止し、関連するVOD出力を作成しています。
    • 完了 -ライブ配信が完了しました
    • キャンセル -ライブストリームが停止しており、保留中の VOD 出力は処理されません。
    • C anceled-ライブ配信がキャンセルされました
    • D@@ isconnected ：エンコーダが切断されました。
    • Deactivated -定期的なイベントが非アクティブ状態にある
    • Not Activated -定期的なイベントはアクティブ化されていますが、ストリーム開始は発生していません
    • W aiting：RTMP ストリームを待機しています（エンコーダが起動します）。
    • 失敗 -システムエラーの結果としてジョブが停止しました
    • 放棄された -エンコーダが接続されていないイベントが終了しました
  • 完了日時
  • イベント名と ID
  • 総ビュー数
  • 期間
  • 作成された日時（定期的なイベントのみ）

ライブイベントのビデオプロパティを表示するには、イベント名をクリックして Control Roo mを開きます。

新しいイベントの作成

Live モジュールを使用したライブイベントの作成とストリーミングに関する詳細な手順については、「 ステップバイステップ:Telestream Wirecast での Live モジュールの使用 」または「 ステップバイステップ:Open Broadcaster ソフトウェア (OBS) での Live モジュールの 使用」を参照してください。

新しいイベントを作成するには、次の手順を実行します。

  1. [イベントの管理] ページで、[ イベントの作成 ] をクリックします。 「ライブイベントの作成」または「ライブチャンネルの作成」ページが開きます。 イベントとチャンネルを作成するときのプロパティは同じです。 このドキュメントのスクリーンショットでは、イベントの作成について説明します。
  2. イベントの詳細を入力します。
    • イベント名 -イベントの名前。ビデオがビデオアセットとして保存されている場合（最大255文字）、この名前はMediaモジュールにも表示されます。
    • 簡単な説明 -イベントの説明（最大250文字）
    • T ags-イベントに関連付けるタグ。ビデオがビデオアセットとして保存されている場合（最大 128 文字と 1200 タグ）、タグがビデオに割り当てられます。
    • 参照ID ：イベントのリファレンス ID。一意である必要があります（最大150文字）
  3. アカウントの必須カスタムフィールドが表示されます。 カスタムフィールドに値を割り当てます。
  4. 「 ソーシャルメディアへのストリーミング 」を「 許可 」に設定すると、イベントをソーシャルメディアプラットフォームにストリーミングできるようになります。 これは、 イベントがストリーミングを開始した後、 または Social モジュール を使用して実行できます。
  5. ライブ取り込みプロファイル を選択します。 ドロップダウンリストには、すべてのライブ取り込みプロファイルが一覧表示されます。 取り込みプロファイルは、作成されるライブレンディションの品質と数を制御します。 「 プロファイルの管理 」リンクをクリックすると、別のブラウザタブにインジェストプロファイルエディタが開きます。 取り込みプロファイルエディタにアクセスするには、アカウント管理者である必要があります。 [ RENDITIO NS] リンクをクリックすると、選択した取り込みプロファイルを使用して作成されるレンディションが表示されます。
    3 つの階層のビットレートと課金の詳細は次のとおりです。
    • ティア1は最大5,500Kbpsで、充電は1.0x時間
    • ティア2は最大10,000Kbpsで、1.5倍の時間で課金されます。
    • 階層 3 は最大で 20,000 Kbps で、充電は 2.0 x 時間です。
    ライブ取り込みプロファイルの作成については、「 カスタムのライブ取り込みプロファイルの作成 」を参照してください。
  6. [ 完了したらイベントをビデオアセットに変換 ] を選択して、ライブビデオの VOD バージョンを Video Cloud アカウントに保存します。 このオプションを選択すると、選択した ライブ取り込みプロファイルの最高 品質のレンディションが VOD ビデオのソースとして使用されます。 レンディションの作成時には、アカウントのデフォルトの取り込みプロファイルが使用されます。 なお、VOD版はライブイベント終了後すぐには提供されず、取り込み処理が完了すると映像が公開される。
    このオプションは、定期的なイベントを使用する場合はサポートされません。
  7. [ 詳細オプション ] をクリックし 、詳細オプションを設定します 。
  8. 「 イベントを作成 」をクリックして、ライブイベントを開始します。 [ Control Room] ページ が開き、エンコーダが使用する ストリーミングエンドポイント（RTMP URL） や ストリーム名 など、ライブイベントの詳細が表示されます。
    ストリームの処理には約 1 分かかります。

    デフォルトでは、新しいイベントが作成されてから 30 分以内にエンコーダを起動する必要があります。

[ ストリーミングを停止 ] ボタンをクリックして、イベントを終了します。

注：ライブ配信の完了時に、エンコーダが閉じているか停止していることを確認してください。 データが送信されていない場合でも、エンコーダとブライトコーブ間の接続を開いたままにしておくと、予期しない接続料金が発生する可能性があります。

詳細オプションの構成

新しいイベントを作成するときに、詳細オプションを設定できます。 詳細オプションを設定するには、イベントの作成時に [ 詳細オプション ] リンクをクリックします。

次のオプションを設定できます。

VODクリップの場合、通知は送信されません。
カスタムエッジ CDN ホスト名をご希望の場合は、ブライトコーブサポートにお問い合わせください。
  • [ Reconnect Time ]：エンコーダが切断されている場合、ライブストリームが使用可能になるまでの時間です。 この設定は、新しいイベントが作成されたときにエンコーダを起動する必要がある時間には影響しません。
  • サーバー側広告の挿入を有効にする (SSAI) -サーバー側広告の挿入を有効にします。 広告は、 サーバー側の広告設定 の一部として構成されます。 Live モジュールでのサーバーサイド広告の操作については、「Live モジュール でのサーバーサイド広告の実装」を参照してください。
    • 塗りつぶしスレート : 塗りつぶしスレートを選択します。 フィルスレートは、ライブ配信の広告区切りで、動的に配信される広告では埋められないギャップがある場合に表示されます。 ライブ設定 の一部としてフィルスレートを作成します。
    • 広告音声 -オンにすると、設定した調整に基づいて広告の音声レベルを正規化します。 これは、広告がコンテンツよりも大きく大きくならないようにするための手段として使用できます。
  • クラウドDVRの作成 -DVR機能を有効にします。 6.33.0 より前のバージョンの Brightcove Player を使用してイベントを公開する場合は、 ライブ DVRUX プラグイン が必要です。 コンテンツは、イベント終了後7日間ご利用いただけます。
    • DVR ウィンドウ -ライブの DVR マニフェストに保持する時間（秒単位）。
  • In put Format-Live モジュールに送信されるエンコーダからの入力フォーマット。 この機能により、今のようにライブストリームで追加のメタデータが可能になり、字幕やキャプション、オーディオトラックの柔軟性が向上し、標準的な放送レベルのデータでデジタル体験が合理化されます。 次の入力形式がサポートされています。
    • RTMP（デフォルト）
    • リアルタイム転送プロトコル（RTP）
    • 安全な信頼性の高い輸送（SRT）
  • 使用する RTMP クレデン シャル-ストリーミングエンドポイントのセキュリティ保護に使用される RTMP クレデンシャルを選択します。 このオプションは、入力フォーマットがRTMPで、少なくとも1つのRTMPクレデンシャルが ライブ設定 の一部として作成された場合にのみ表示されます。
  • RTMP IP ホワイトリスト -RTMP エンドポイントへの接続を許可する IP アドレスのセットを入力します。 これは、RTMP エンドポイントに接続できるユーザーを制御します。 このオプションは、入力フォーマットがRTMPの場合にのみ表示されます。
  • CIDR ホワイトリスト -RTP または SRT エンドポイントへの接続を許可する IP アドレスのセットを入力します。 これは、RTP エンドポイントまたは SRT エンドポイントに接続できるユーザーを制御します。 このオプションは、入力フォーマットが RTP または SRT の場合にのみ表示されます。
  • リージョン -使用するアマゾンウェブサービス（AWS）リージョン。 レイテンシーを減らすには、エンコーダが配置されている場所に最も近いリージョンを選択する必要があります。 デフォルトでは、リージョンは Oregon (米国西部) に設定されており、別の大陸からストリーミングしている場合、悪い結果になります。 次のリージョンがサポートされています。
    • オレゴン (米国西部) (デフォルト)
    • バージニア (米国東部)
    • 東京（アジア太平洋）
    • シンガポール（アジア太平洋）
    • シドニー（アジア太平洋）
    • ムンバイ（アジア太平洋）
    • フランクフルト (EU)
    • アイルランド (EU)
  • コンテンツ暗号化 -ライブストリームに適用する暗号化の種類を選択します。
    • なし ：暗号化は適用されません。
    • AES-128 -AES-128 暗号化を使用してストリームを保護します。 このオプションを選択した場合は、 暗号化キー を入力する必要があります。 キーは 32 文字の 16 進数である必要があります。 最良の結果を得るには、 128 ビット と 16 進数の オプションを選択して、 キージェネレータ を使用します。
      暗号化されたライブストリームからクリップを作成することはできません。
    • デジタル著作権管理（DRM）-デジタル著作権管理テクノロジーによるセキュアなプレミアムコンテンツストリームです。 詳細については、 ライブ DRM ドキュメントを参照してください。
      • HLS + フェアプレイ
      • HLS（fMP4）+ ワイドバインモジュラー
      DRM を使用するには、ブライトコーブサポートが最初に Live アカウントを有効にする必要があります。 DRM 対応イベントを公開するには、DRM 対応プレーヤー v6.37.1 以降を作成して使用する必要があります。
  • ストリームステータス通知を有効にする -オンにすると、 通知URL を入力します。 ジョブのステータスが変更されると、 通知URL にhttpリクエストが行われます。 これは、ジョブのステータスの変化に応じてサーバーを更新するために使用できます。 API 通知の使用方法の詳細については、「 ライブ API 通知 」トピックを参照してください。
  • + マルチ CDN 設定を追加 -追加の CDN の ラベル と URL を入力します。 これは、主要なCDNに動画セグメントを尋ねることが許可されている CDN のホワイトリストです。 セカンダリ CDN を正しく設定する責任があります。
  • 静的エントリポイント（SEP）を使用してこれを定期的なイベントとして 作成する-単一のライブストリームを作成し、必要に応じてアクティブ化/非アクティブ化できるようにします。 詳細については、このドキュメントの「 定期的なイベントの概要 」セクションを参照してください。

[ Edit Configuration ] をクリックして、ライブストリームを作成するために作成される API リクエストを表示/編集します。 API 呼び出しを直接編集する場合は、注意が必要です。

定期的なイベントの概要

[ 静的エントリポイント (SEP) を使用してこれを定期的なイベントとして作成する] オプションを選択すると、同じエントリポイント の URL と再生 URL を維持したまま、長時間実行されるライブストリームをアクティブ化および非アクティブ化することができます。 定期的なイベントを使用する場合、ストリーミングエンドポイント URL とビデオ ID は変更されません。

定期的なイベントを作成すると、Media モジュールでそのイベント用のビデオが自動的に作成されます。 イベントが再発するたびに必要になるため、このビデオを削除 しないでください 。

制限事項

定期的なイベントの操作には、次の制限が適用されます。

  • [ 完了時にイベントをビデオアセットに変換 ] オプションは使用できません。 代わりに、クリッピングを使用して、完全なライブイベントを VOD としてアーカイブすることもできます。 クリップは、ストリームの次回のアクティブ化まで、または 7 日間のいずれか短い方で作成できます。 ストリームが再びアクティブになると、以前の録画セッションはサーバーから削除され、クリップできなくなります。
  • ジョブ設定は変更できません。現在、繰り返しイベントを最初に作成した設定に関連付けられています。
  • ストリームをアクティブ化すると、イベントが使用可能になるまでに最大 60 秒かかることがあります。 一部のエンコーダにはリトライ（Wirecast、Elemental など）が組み込まれているため、エントリポイントが利用可能になるとすぐにエンコーダが再接続されます。
  • [ 再接続時間] は、エンコーダの切断時に繰り返し発生するイベントにも適用されます。 再接続時間 に達すると、定期的なイベントは非アクティブ状態に設定されます。 Brightcove では、ジョブの完了時にストリームを無効にし、切断時に 再接続時間 （Reconnect Time）を許容可能な時間に設定することをお勧めします（この待機期間中に入力時間のコストが発生します）。
  • 定期的なイベントを非アクティブ化する前に、エンコーダを常に停止する必要があります。 アカウントには、一定数の繰り返しイベントが割り当てられます。 制限に達した場合は、新しいイベントを作成する前に、定期的なイベントの 1 つを停止する必要があります。 繰り返し発生するイベントの最大数に達していて、別のイベントを作成しようとすると、エラーが返されます。

定期的なイベントの操作

定期的なイベントを作成するには、ライブイベント の作成時に [静的エントリポイント (SEP) を使用してこれを定期的なイベントとして 作成] オプションを選択し、[ イベントを作成 ] をクリックします。

定期的なイベントが作成されると、そのイベントは [イベントの管理] ページの [ スタンバイ ] タブに表示されます。

定期的なイベントステータスは、次のいずれかになります。

  • Not Activated -定期的なイベントが作成され、ストリーミングが開始されたことがない
  • W aiting-定期的なイベントがアクティブになり、ストリームを待機しています
  • ストリーミング -進行中の定期的なイベント
  • 切断 -定期的なイベントが停止中です
  • Deactivated -定期的なイベントが非アクティブになっています。
  • アーカイブ済み -定期的なイベントがアーカイブされました

定期的なイベントのアクティブ化

定期的なイベントを有効にすると、ジョブをストリーミングで使用できるようになります。 recurring イベントは、エンコーダが起動される前にアクティブにする必要があります。 定期的なイベントをアクティブにするには、次の手順に従います。

  1. [ スタンバイ ] タブをクリックします。
  2. イベント名をクリックすると、コントロールルームでイベントが開きます。
  3. [ アクティブ化 ] をクリックします。
  4. エンコーダを起動します。

定期的なイベントがアクティブになると、[ スタンバイ ] タブから [ ライブ ] タブに移動します。

定期的なイベントの非アクティブ化

定期的なイベントを非アクティブにすると、ジョブのストリーミングが無効になります。 イベントが終了すると、定期的なイベントを非アクティブにする必要があります。 定期的なイベントを非アクティブ化するには：

  1. [ ライブ ] タブをクリックします。
  2. イベント名をクリックすると、コントロールルームでイベントが開きます。
  3. [ 非アクティブ化 ] をクリックします。
  4. エンコーダを停止します。

定期的なイベントのアーカイブ

定期的なイベントをアーカイブすると、ジョブが永久に非アクティブになり、ストリーミングを有効にできなくなります。 最終的にイベントは削除されます。 ストリーミングが開始されたイベントのみをアーカイブできます。 定期的なイベントが作成され、ストリーミングが開始されない場合は、[ キャンセル ] ボタンと [ アーカイブ ] ボタンが表示されます。

定期的なイベントをアーカイブするには：

  1. [ スタンバイ ] タブをクリックします。
  2. イベントの行をクリックして選択します。
  3. [ アーカイブ ] をクリックします。

コントロールルームを使用したライブイベント情報の表示

「コントロールルーム」ページは、ライブイベントに関する情報を表示するために使用します。 新しいライブイベントを作成してストリーミングを開始すると、Control Room ページが開き、イベント情報が表示されます。 プレビュープレーヤーにイベントのプレビューが表示されます。 「コントロールルーム」ページは、「ライブイベントの管理」ページでイベントをクリックしたときにも表示されます。 プレビュープレーヤーの下にある [ Monit oring] タブをクリックして、ストリームの進行中に モニタリング情報を表示します 。

[コントロールルーム] ページの [ イベント情報 ] タブは、次のセクションで構成されています。

  • イベント・ステータス
  • イベント情報
  • クリップ
  • レンディション
  • RTMP
  • ソーシャルイベント

プレビュープレーヤーの下にある [ Monit oring] タブをクリックして、ストリームの進行中に モニタリング情報を表示します 。

イベント・ステータス

[Event Status] セクションは、イベントの進行中のみ表示されます。 次の情報が表示されます。

  • ジョブ・ステータス ：現在のジョブ・ステータス
    • ストリーミング -ストリームはライブです
    • 処理エラー -ライブストリームでエラーが発生しました
    • Standby ：ストリーム起動準備完了（静的エントリー・ポイントのみ）
    • Stopp@@ ing -ライブストリームが停止し、関連するVOD出力を作成しています。
    • 完了 -ライブ配信が完了しました
    • キャンセル -ライブストリームが停止しており、保留中の VOD 出力は処理されません。
    • C anceled-ライブ配信がキャンセルされました
    • D@@ isconnected ：エンコーダが切断されました。
    • W aiting：RTMP ストリームを待機しています（エンコーダが起動します）。
    • 失敗 -システムエラーの結果としてジョブが停止しました
  • 現在のビューア -現在のビューアの数。 これは単なる推定値であり、視聴者の数が少ないイベントや視聴者の急増などでは、数値は正確ではない可能性があることに注意してください。
  • [ Elapsed Time]： 開始 されてからの経過時間をストリームします（作成時ではありません）。
  • 現在の時刻- 現在のUTC時刻

イベント情報

「イベント情報」セクションには、次の情報が表示されます。

  • Name ：イベントの名前
  • ジョブ ID -内部ジョブ ID
  • ビデオ ID
  • 簡単な説明
  • [ タグ]
  • 参照ID
  • ソーシャルメディアへのストリーミング - 許可 されている場合、ライブイベントをスケジュールされたソーシャルイベントに割り当てることができます
  • ストリーミングエンドポイント（RTMP URL ）：エンコーダが使用するストリーミングエンドポイント
  • Stream Name ：エンコーダが使用するストリーム名
  • リージョン -ストリーミングリージョン
  • 再生 URL -非 BrightCove Player でライブストリームを表示するために使用できる URL

[クリップ]

Clips セクションには、ライブストリームから作成されたすべてのクリップが表示されます。 クリップ名をクリックすると、Media モジュール内のクリップが開きます。 クリップの作成については、「 Live モジュールを使用したビデオクリップの作成 」を参照してください。

レンディション

[レンディション] セクションには、作成されたレンディションの詳細が表示されます。 レンディションのリンクをクリックすると、レンディションのエンコーディング設定が表示されます。

RTMP

RTMP セクションを使用して、RTMP 出力を追加できます。 これにより、ライブストリームを他の RTMP エントリポイントに送信できます。 これは、FacebookやYouTubeにストリーミングするためにライブを送信するために使用することができます。

RTMP 出力 URL と送信する レンディション を入力します。 RTMP ストリームが使用されている場所によって、出力 URL の形式が変わる場合があることに注意してください。 詳細については、「 ライブ RTMP 出力を使用して Facebook や YouTube にストリーミングする 」を参照してください。

ソーシャルイベント

[ソーシャルイベント] セクションには、ライブストリームが割り当てられている スケジュールされたソーシャルイベント に関する情報が表示されます（存在する場合）。

イベントの監視

プレビュープレーヤーの下にある [ Monitoring ] タブをクリックして、モニタリング情報を表示します。 監視情報は、イベントの進行中およびイベントの終了後最大 1 時間まで使用できます。

デフォルトでは、モニタモジュールは出力ドリフトと入力ドリフトのグラフのみを表示します。 モニターの拡張版をご希望の場合は、ブライトコーブのアカウント担当者にお問い合わせください。

左側のナビゲーションで [ レポート ] をクリックして、 ライブ分析 を表示します。

[ Select Graph s] をクリックして、表示するグラフを選択し、各グラフの内容の説明を選択します。

イベントの複製

イベントを複製すると、以前のライブイベントの作成に使用したものと同じ設定で別のイベントを簡単に作成できます。 イベントを複製するには、次の手順に従います。

  1. イベントをクリックして「コントロールルーム」ページを開きます。
  2. ページ上部の [ イベントを複製 ] をクリックします。 前のライブイベントの設定を使用して、新しいライブイベントが作成されます。 必要に応じて、イベントのプロパティを変更します。
  3. [ イベントを作成 ] をクリックします。

ライブイベントの埋め込みコードの生成

イベントの埋め込みコードを生成するには、左側のナビゲーションで [ パブリッシュと埋め込み] > [プレーヤーの設定 ] をクリックします。 次の 2 種類の埋め込みコードを使用できます。

  • 標準 -iframe埋め込みコード。 ほとんどの場合に推奨されます。 このタイプは、プレーヤーを<iframe> タグ内に配置します。
  • 詳細設定 -ページ内埋め込みコード。 プレーヤーがiframeではなく、HTMLページに直接存在することを許可します。 これにより、開発者には、プレーヤーや関連するプロパティやイベントへのアクセスが容易になるという利点があります。

また、公開されたプレーヤーの URL を取得することもできます。 これは、ブラウザでイベントを表示する場合に便利です。

ライブイベントの埋め込みコードを生成するには、次の手順に従います。

  1. イベントをクリックして「コントロールルーム」ページを開きます。
  2. 左側のナビゲーションで [ パブリッシュと埋め込み] > [プレーヤーの設定 ] をクリックします。
  3. プレーヤー を選択します。
    注:サーバーサイド広告（SSAI）、DVR、またはマルチ CDN を有効にするようにイベントが設定されている場合は、Brightcove Player バージョン 6.18.3 以降を使用する必要があります。 バージョン6.18.3以上のプレイヤーのみがドロップダウンリストに表示されます。
  4. [ DVRを許可 ] をオンにすると、プレーヤーはライブマニフェストではなくDVRマニフェストを要求します。 このオプションは、[ Create Cloud DVR ] オプションを選択してイベントを作成した場合にのみ使用できます。 6.33.0 より前のバージョンの Brightcove Player を使用してイベントを公開する場合は、 ライブ DVRUX プラグイン が必要です。
    注：
    • 「DVRを許可」機能は、現時点ではギャラリーモジュールと互換性がありません。
    • 「DVRを許可」オプションを選択すると、その値は ADConfigId=Live に変更されます。
  5. Player の URL をクリックしてブラウザでイベントをプレビューするか、適切な埋め込みコードをコピーして使用します。

    ライブストリーミングをプレビューするもう 1 つのオプションは、API を使用して行うことです。

    Live API を使用して Live ジョブを作成したときに返された playback_url 値をコピーし、Safari ブラウザに貼り付けます。 このブラウザはネイティブの HLS 再生をサポートしているため、ライブストリームが表示されます。

    また、URL をコピーして、ブライトコーブプレーヤーリファレンスアプリに貼り付けることができます。

    https://players.brightcove.net/3303963094001/player-reference-app/index.html

ソーシャルメディアへの接続

ライブイベントを、Social モジュールでスケジュールされたソーシャルイベントに接続できます。 ライブイベントをソーシャルイベントに接続する前に、Social モジュールでライブイベントを作成する必要があります 。 「 ソーシャルメディアへのストリーミング 」設定を「 許可 」に設定して作成されたイベントのみを、スケジュールされたイベントに割り当てることができます。

ライブイベントをソーシャルイベントに接続するには、次の手順に従います。

  1. Social モジュールでライブイベントを作成します 。
  2. イベントをクリックして「コントロールルーム」ページを開きます。
  3. 左側のナビゲーションで「 公開と埋め込み」>「ソーシャルメディアに接続 」をクリックします。 スケジュールされたソーシャルイベントのリストが表示されます。 イベントが表示されない場合、スケジュールされたイベントは存在しません。
  4. スケジュールされたイベントをクリックし、[ 接続 ] をクリックします。

ライブモジュールを使用してイベントが割り当てられたら、Social モジュールに戻ってイベントをプレビューし、ソーシャルプラットフォームへのストリーミングプロセスを開始します。

クリップの作成

クリップは、イベントの進行中またはイベントの終了後に作成できます。 クリップを作成できない場合は、左側のナビゲーションで [ クリップの作成] にカーソルを合わせるとメッセージが表示されます。 クリップの作成については、「 Live モジュールを使用したビデオクリップの作成 」を参照してください。

注：

  • ライブイベントの終了後 7 日間は、ストリームをクリッピングできます。

  • 静的エントリポイント (繰り返し) ストリームの場合:ストリームの 7 日後、ストリームが 2 番目のストリームで再アクティブ化される場合、クリッピングは 2 番目のストリームのコンテンツに対してのみ使用できます。 最初のストリームはその時点でクリアされ、ストリームが非アクティブ化されてから 7 日以内であっても、そこからクリップを作成する方法はもうありません。

広告のリクエスト

イベントの作成時に [ サーバー側広告の挿入を有効にする (SSAI) ] オプションが選択されている場合は、コントロールルームのページに [ 広告の移動 ] ボタンが表示されます。 クリックすると広告枠の長さを選択できます。

[ 挿入 ] をクリックして、ライブ配信に広告区切りを挿入します。 広告が広告の休憩時間全体を満たさない場合は、塗りつぶしスレタイが表示されます。 Live モジュールでのサーバーサイド広告の操作については、「Live モジュール でのサーバーサイド広告の実装 」を参照してください。

キャプション

ブライトコーブライブは、現在 608 個のキャプションをインバンド（h.264 ヘッダー内）でサポートしています。 キャプションが h.264 入力信号 (user_data パケットで正しくシグナリング) 内にある場合、キャプションは h.264 出力に渡されます。

放送エレメンタルライブエンコーダを使用している場合は、SDI（EIA-608/CEA-608）または他のソース（SCTE-20、SCC、文字多重放送、DVB-Sub、補助的、ARIB、TTML、SCTE-27、STL、SRT、SMI）からキャプションを取得し、私たちに送信する h.264 ストリームに入れることができます。 他の放送グレードのエンコーダもおそらく同じことを行うことができますが、正式にはテストしていません。

WebVTT キャプションは、ライブストリームではサポート されていません 。

Brightcove Player では、帯域内キャプションがサポートされています。

ライブ配信に関する問題のトラブルシューティング

ライブイベントを動作させるためにさらにサポートが必要な場合は、 当社までご連絡ください 。 可能な限り最速の応答が得られるように、問題を解決するために必要なサポートのリストを以下に示します。

  • ストリームに発生している特定の症状。 例えば、それはまったく再生されないのですか、それは吃音またはフリーズしますか？
  • このストリームが過去に正しく機能したかどうか
  • エンコーダで使用しているエントリポイント URL
  • エンコーディングソフトウェアとハードウェアは、
  • ライブイベントを公開したプレイヤーの URL
  • Video Cloud Studio でのライブアセットのビデオ ID
  • エンコーダからパブリッシングポイントホストへのトレースルートの結果
