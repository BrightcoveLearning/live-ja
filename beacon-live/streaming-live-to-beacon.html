ビーコンアプリへのライブイベントのストリーミング

このトピックでは、ビーコンアプリにライブイベントをストリーミングする方法について説明します。

概要

ライブイベントをアプリケーションにストリーミングするには、一連の手順に従って Video Cloud Live アカウントと Brightcove Beacon を接続する必要があります。

プロセスの概要は、次のとおりです。

  1. まず、ブライトコーブビーコンでイベントとプレイリストを作成します。 このイベントは、あなたのビデオのプレースホルダーとしてアプリで利用できるようになります。
  2. Video Cloud アカウントに移動して Live にアクセスしてイベントを作成し、ID として機能するジョブ ID を取得します。
  3. Brightcove Beacon に戻ると、ライブプラットフォームで生成されたジョブ ID をプラットフォームの [イベント] セクションの下に置いて、ライブストリーミングを接続することができます。
  4. 接続を待つと、ビーコンアプリでライブビデオのストリーミングが開始されます。
  5. 最後のセクションでは、静的エントリポイントによるライブで異なるタイプのイベントを作成する手順が表示されます。

このドキュメントの残りの部分では、これらの各手順について詳しく説明します。

ビーコンでライブイベントを作成します。

Brightcove Beacon プラットフォームで、[ Events ] タブをクリックして、イベントのセットアップを開始します。

[タブ] セクションで プレイリスト と イベント を利用できるのは、 Brightcove Beacon Live でアクティブ化されたアカウントのみです。 アクティベーションを行っていない場合は、ブライトコーブサポートにお問い合わせください。
  1. [ 新しいイベントを追加 ] をクリックします。
  2. ライブイベントに 名前と イベント開始日を指定します 。
  3. [ テキストデータ ] タブで、イベント情報を入力します。
  4. [ 非テキストデータ ] タブで、イベントのイメージポスターを選択します。

    イベントに使用するポスターの種類は、ポートレート、風景、サムネイル、ワイドバナーから選択できます。

  5. [ 権利とスケジュール ] タブでは、 可用性の詳細 を次のように設定できます。
    • 権利の種類: [広告]、[サブスクリプション]、[無料] のいずれかの権利の種類を選択します。
    • 開始時間: イベントの開始時間
    • 終了時間： イベントの終了時間。これらの時間は、アプリ内で制限を生成するために作成されます。
    • 最大ストリーム： アプリで許可されるストリームの最大数です。 デフォルトの数値は 0 (ゼロ) です。
    • デバイスの詳細: ここでは、ストリームを公開する別の数のデバイスを追加できます。 （例：iOS、Android、Roku、サムスンなど）
    • 地理的位置情報の詳細: アプリを有効にする国の一覧です。 どの国も制限したくない場合は、「世界」を選択できます。
    • 拒否された場所: コンテンツへのアクセスが拒否された地域の一覧。
  6. 下のボタンをクリック: 新しいイベントを作成

プレイリストへのビーコンイベントの追加

イベントを作成したら、上部メニューの [ プレイリスト ] タブに移動します。 何かをストリーミングする前に、イベントをプレイリストに追加する必要があります。

  1. [ 新しいプレイリストを追加] ボタンをクリックし、プレイリスト を作成します。
    ライブイベントを含むプレイリストを Brightcove Beacon で作成する必要があります。 Video Cloud のプレイリストを使用@@ しないでください 。Video Cloud のプレイリストを使用すると、Beacon Live 録音が上書きされ、削除されることがあります。
  2. [ 再生リスト ] の [コンテンツタイプ] タブから、[ イベント ] タブを選択します。
  3. コンテンツモジュールでイベントを探し、（+）ボタンをクリックして左側のプレイリストに追加します。 イベントの順序は、アプリに表示したいとおりに配置できます。
  4. プレイリストの設定を調整する（時間、デバイス、場所など） あなたのイベントで行ったように。
  5. [ プレイリストに変更を保存 ] ボタンをクリックします。
  6. 画面右上のツールボタンをクリックしてキャッシュをクリアすると、アプリにすべて正しく表示されます。

この時点で、イベントはBrightcove Beacon アプリのコンテナとして作成されますが、実際にライブ動画のストリーミングを開始するには、Video Cloud Live Platform で送信を設定する必要があります。

スタジオでライブイベントを作成する

ライブモジュールにアクセスするには、Video Cloud Studio にログインし、ナビゲーションヘッダーの [ライブ] をクリックします。

  1. [イベントの管理] ページで、[ イベントの作成 ] をクリックします。 「ライブイベントの作成」または「ライブチャンネルの作成」ページが開きます。 イベントとチャンネルを作成するときのプロパティは同じです。 このドキュメントのスクリーンショットでは、イベントの作成について説明します。
  2. イベントの詳細を入力します。
    • イベント名 -イベントの名前。ビデオがビデオアセットとして保存されている場合（最大255文字）、この名前はMediaモジュールにも表示されます。
    • 簡単な説明 -イベントの説明（最大250文字）
    • T ags-イベントに関連付けるタグ。ビデオがビデオアセットとして保存されている場合（最大 128 文字と 1200 タグ）、タグがビデオに割り当てられます。
    • 参照ID ：イベントのリファレンス ID。一意である必要があります（最大150文字）
  3. アカウントの必須カスタムフィールドが表示されます。 カスタムフィールドに値を割り当てます。
  4. 「 ソーシャルメディアへのストリーミング 」を「 許可 」に設定すると、イベントをソーシャルメディアプラットフォームにストリーミングできるようになります。 これは、 イベントがストリーミングを開始した後、 または Social モジュール を使用して実行できます。
  5. ライブ取り込みプロファイル を選択します。 ドロップダウンリストには、すべてのライブ取り込みプロファイルが一覧表示されます。 取り込みプロファイルは、作成されるライブレンディションの品質と数を制御します。 「 プロファイルの管理 」リンクをクリックすると、別のブラウザタブにインジェストプロファイルエディタが開きます。 取り込みプロファイルエディタにアクセスするには、アカウント管理者である必要があります。 [ RENDITIO NS] リンクをクリックすると、選択した取り込みプロファイルを使用して作成されるレンディションが表示されます。
    3 つの階層のビットレートと課金の詳細は次のとおりです。
    • ティア1は最大5,500Kbpsで、充電は1.0x時間
    • ティア2は最大10,000Kbpsで、1.5倍の時間で課金されます。
    • 階層 3 は最大で 20,000 Kbps で、充電は 2.0 x 時間
  6. [ 完了したらイベントをビデオアセットに変換 ] を選択して、ライブビデオの VOD バージョンを Video Cloud アカウントに保存します。 このオプションを選択すると、選択した ライブ取り込みプロファイルの最高 品質のレンディションが VOD ビデオのソースとして使用されます。 レンディションの作成時には、アカウントのデフォルトの取り込みプロファイルが使用されます。 なお、VOD版はライブイベント終了後すぐには提供されず、取り込み処理が完了すると映像が公開される。
    注：このオプションは、定期的なイベントを使用する場合はサポートされません。
  7. [詳細オプション] をクリックし、詳細オプションを設定します。
    注意：ビーコンライブ作業の場合は、詳細オプションの「ストリームステータス通知」チェックボックスを有効にする必要があります。
  8. ストリームステータス通知を有効にする

    ビーコン CMS からの通知 URL を追加します（この URL はブライトコーブサポートから提供されます）。

  9. 「 イベントを作成 」をクリックして、ライブイベントを開始します。 [Control Room] ページが開き、エンコーダが使用する ストリーミングエンドポイント（RTMP URL） や ストリーム名 など、ライブイベントの詳細が表示されます。

    ストリームの処理には約 1 分かかります。

    Live プラットフォームでストリーミングを開始するには、エンコーダが必要です。エンコーダの使用方法については、次のドキュメントを参照してください。

    ステップバイステップ:ライブモジュールとテレストリーム Wirecast を使用する

    ステップバイステップ：オープンブロードキャスタソフトウェア（OBS）でのライブモジュールの使用

  10. コントロールルーム で、 テストイベント パネルから 「ジョブID」 と「ビデオID」 をコピーし、ビーコンプラットフォーム設定でライブイベントを設定する必要があります。
  11. 次に、ビーコンに戻り、[イベント] タブをクリックし、以前に作成したイベントに移動し、[基本データ] 設定でライブイベントのジョブ ID を貼り付けます。

    次に、[ ストリーム ] タブに移動して新しいストリームを作成し、[ ストリームビデオ ID ] に、前に作成した Video Cloud イベントからビデオ ID を貼り付けます。

    次の情報を入力します。

    • ビデオを公開できるデバイスの数を選択します。
    • [ ストリームの種類] を [ ライブ ] に変更します。
    • ページを離れる前に 「イベントの更新」 ボタンをクリックします：
  12. キャッシュをクリアします。

    ブライトコーブビーコンアプリに移動し、プレイリストとイベントを選択します。 [ 再生 ] をクリックします。

    これで、Beacon アプリで Video Cloud からのライブストリーミングを見ることができます。

    イベントが終了したら、クリップとして保存し、後でビーコンのVODとして送信できます。

ビーコンライブイベントからクリップを作成する

後でVOD録画ビデオとしてストリーミングするためにビデオを保持したい場合は、ライブビーコンイベントのクリップを作成する必要があります。

Live モジュールは、完了したイベントからビデオクリップを作成する機能を提供します。 ライブイベントの進行中にクリップを作成することもできます。

クリッピング機能の完全なリストについては、「 Live モジュールを使用したビデオクリップの作成」の ドキュメントを参照してください。

  1. Live モジュールで、左側のパネルの [ クリップを作成 ] タブに移動します。
  2. クリッピングボタンまたはショートカットキーを使用して、クリップの開始位置と終了位置を設定します。 または、ビデオ全体を保持するには、[ 保存] ボタンをクリックしてください 。
  3. クリップビデオのプロパティを入力します。
    • Clip Name ：クリップの名前
    • 短い説明 -クリップの説明
    • タグ -クリップに関連付けるタグ
    • リファレンス ID -クリップに割り当てるリファレンス ID
    • ソース広告区切りコンテンツを削除 -このオプションを選択すると、プログラムコンテンツのみを保持し、クリップ内で発生した広告区切りを削除します。 注:[ソース広告の区切りを削除] チェックボックスは、ストリームが SSAI が有効になっている場合にのみ表示されます。
  4. [ クリップを作成 ] をクリックします。 クリップのトランスコーディング処理が開始されます。 クリップは、アカウントのデフォルトの取り込みプロファイルを使用してトランスコードされます。

再生可能なストリームビデオとしてクリップをビーコンに追加します。

  1. 作成したクリップの番号 ID を取得します 。
  2. Beacon プラットフォームで、[ イベント ] タブに移動して [ ストリーム ] メニューを選択し、ドロップダウンで [ ストリームタイプ ] を [ライブ] から [ストリーム] に変更します。次に、[ ビデオ ID をストリー ミング] オプションで、クリップイベントからコピーした ID 番号をライブプラットフォーム。
  3. [ 基本データ ] タブをクリックし、[ ステータス ] を [ 終了 ] から [ 完了 ] に変更します。 終了する前に、ページの下部にある [ イベントの更新 ] ボタンをクリックすると、イベントの記録を確認するためにアプリの [ 再生 ] ボタンが利用できるようになります。

(静的エントリポイント) SEP ライブイベント

SEP は定期的なイベントであり、毎週のプログラムなど、イベントを繰り返し開始したときにアクティブ/非アクティブにすることができます。 このイベントは、ビーコンアプリ内の複数のチャネルに送信できるすべての ライブジョブID のイベントとして使用できます。

  • 定期的なイベントを作成すると、Media モジュールでそのイベント用のビデオが自動的に作成されます。 イベントが再発するたびに必要になるため、このビデオを削除しないでください。
  • recurring イベントは、エンコーダが起動される前にアクティブにする必要があります。

定期的なイベントを作成するには、ライブイベント の作成時に [静的エントリポイント (SEP) を使用してこれを定期的なイベントとして 作成] オプションを選択し、[ イベントを作成 ] をクリックします。

  1. ライブプラットフォーム で新しいイベントを作成します 。 イベントの作成方法の詳細については、次の ドキュメントを参照してください 。
  2. [ 詳細オプション ] を展開し、[ 静的エントリポイント (SEP) を使用して定期的なイベントとして作成する] オプションを有効にします。 SEP の詳細については、 こちらをクリックしてください。
  3. SEP イベントが既にある場合は、[ スタンバイ ] タブをクリックすると、Live Control Room からイベント を有効/無効に できます。
  4. ライブジョブ ID をコピーする
  5. Brightcove Beacon プラットフォームに戻り、イベントに名前とスケジュールされた時刻を設定します。
  6. イベントの日付と時刻は、次の形式にする必要があります。
  7. イベントを更新 ボタンをクリックします。

このプロセスは、Brightcove Beacon プラットフォームで必要なすべてのイベントチャンネルで実行できます。すべてのチャンネルは、同じライブイベント送信からストリーミングされますが、それぞれのイベント設定で調整した日時が異なります。
