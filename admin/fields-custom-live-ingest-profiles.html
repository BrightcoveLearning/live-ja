カスタムライブ取り込みプロファイルのフィールド

このトピックでは、Live の取り込みプロファイルで使用されるフィールドのリファレンスを示します。

プロファイルフィールド

プロパティ タイプ 説明 デフォルト
account 整数 ビデオクラウド アカウント ID なし
名前 文字列 プロファイル名-アカウント内で一意である必要があります なし
display 文字列 ［Studio Admin］ページに表示されるプロファイルの表示名は、 名前 と同じでもかまいませんが、ユーザーがプロファイルの目的を理解できるように説明する必要があります。 なし
説明 文字列 プロファイルの説明 なし
version 整数 プロファイルをライブモジュールに表示するには、この値を 1 に設定します。 なし
レンディション 配列 レンディションオブジェクトの配列 []
パッケージ＃パッケージ＃ 配列 DRMパッケージの配列-現在、DRMはサポートされていないため、これは常に空の配列でなければなりません []

レンディションフィールド

次の表は、ライブ取り込みプロファイルを更新するためにリクエスト本文で使用できるフィールドを示しています。

プロパティ タイプ 説明 デフォルト
bitrate 整数 オーディオビットレート（kbps） 一致ソース
codec 文字列 オーディオコーデック- aac | mp3 フォーマットまたはプロファイルによって決定される<-ほとんどの場合、aacは最良/td>
bitrate_cap 整数 kbps 単位で、バッファを介してデコーダに供給される最大ビットレート なし
buffer 整数 kbps 単位で、 bitrate_cap を使用するときにデコーダに供給されるバッファのサイズ なし
forced_keyframe 番号 X フレームごとにキーフレームを強制しますが、シーンカットなどに追加のキーフレームを許可します。 なし
フォーマット＃フォーマット＃ 文字列 ビデオフォーマット- TS 、 MP4 、 FLV なし
profile 文字列 H.264 には、 ベースライン (最低)、 メイン 、および 高 の 3 つの一般的に使用されるプロファイルがあります。 [ 1-5] ベースライン
optimized_ts ブール値 HLS 用に最適化された時間セグメント-HLS 出力にのみ適用 なし
keyframe 整数 キーフレーム間の最大フレーム数 250
keyframe フロート 1 秒あたりの最大キーフレーム数 250@@ の間隔
ラベル 文字列 画像レンディションの画像タイプ- ポスター または サムネイル なし
stream ブール値 これがライブビデオストリームになるかどうか なし
dvr_sliding_window 整数 LiveVR 配信のために使用可能にしておくストリームの継続時間（秒） 300
bitrate 整数 kbps で、123 = 123kbps-h264のみ なし
max フロート は フレームレート を設定するのではなく、制限します。 60 FPS のサポートによってアカウントが有効になっていない限り、30 に設定する必要があります。 なし
media_type 文字列 ビデオ または オーディオ なし
bitrate 整数 kbpsで、123=123kbpsで なし
codec 文字列 例：h264, ts フォーマットまたはプロファイルによって決定
幅 整数 フレーム幅 (ピクセル単位) なし
高さ 整数 フレームの高さ (ピクセル単位) なし
