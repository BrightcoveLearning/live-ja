概要:ブライトコーブライブ API

この概要では、Live API の用途とその使用方法を学習します。 このドキュメントに含まれるトピックには、サポートされる AWS リージョンと CDN、ライブチャンネルとイベント、ID3 タイミングメタデータのライブストリームへの挿入が含まれます。

はじめに

Live API は R EST ベースの API で、ライブストリーミングイベントを作成および管理できます。 オプション機能は次のとおりです。

  • サーバーサイド広告挿入（ SSAI ）
  • AES-128 暗号化
  • ライブストリームから取得したクリップからビデオオンデマンドアセットを作成する
  • DVR 機能
  • 複数の CDN

「 API リファレンス 」も参照してください。

ベース URL

ライブ API のベース URL は次のとおりです。

 https://api.bcovlive.io

ヘッダー

すべてのリクエストは、アカウントの設定時に提供される API キーを使用して認証されます。 キーは X-API-KEY ヘッダーで渡されます。 Content-Type ヘッダーも必要です。

 X-API-KEY： あなたの_APIKey 
 コンテンツタイプ：アプリケーション/ json

サポートされている AWS リージョン

次の AWS リージョンがサポートされています。

サポートされる AWS リージョン

バージニア州 us-east-1
東京 ap-norteast-1

SEP ジョブは、標準の制限が 3 のアカウントによって制限されます。ただし、 us-west-2 の制限が最大 10 です。 すべての制限は、地域ごとではなく、アカウントごとに設定されます。

サポートされている CDN

ライブストリーミングでは、次の CDN がサポートされています。

  • アカマイ
  • クラウドフロント

他のファイルベースの CDN は動作しますが、テストされておらず、積極的にサポートされていません。

チャンネルとイベント時間

Live には次の 2 つの購入オプションがあります。

  • ストリーミング時間の購入イベント時間
  • ストリーミングチャンネルを購入する

また、イベントのストリーミング時間とチャンネルの両方を購入することもできます。 オファリングの詳細については、アカウントマネージャーにお問い合わせください。

イベントのストリーミング時間とチャンネルの両方を購入した場合は 、次の点に注意する必要があります。
  • デフォルトでは、ライブジョブはイベントストリーミング時間に対して課金されます。
  • 代わりにジョブをチャネルジョブとして請求する場合は、ライブジョブの作成時にリクエスト本文に次のフィールドを含める 必要があります 。

    「channel_type」:「24x7"

  • チャネルのみの顧客は、ライブジョブを作成するときに、常にリクエスト本文に次のフィールドを含める必要があります。

    「channel_type」:「24x7"

  • ストリーミング時間のイベント時間のみを購入したお客様は、リクエストに 「channel_type」:「24x7 」を含め ないでください 。

トークン認証

ブライトコーブは、ライブビデオストリーム再生 URL にトークン認証を追加するオプションを提供しています。 トークン認証を追加する場合は、 Brightcove サポートにお問い合わせください 。 トークン認証の設定には、最大 3 日間かかる場合があります。

トークンのTTL (存続時間) は、1 時間から 365 日の任意の値に設定できます。 TTL を設定する期間は、デプロイするライブストリームの種類によって異なります。 ただし、TTL はアカウント全体の設定であり、すべてのライブ配信に適用されます。

DVR 機能

ブライトコーブのライブストリームは DVR 機能を備えています。 この機能を使用するには、次の操作を行う必要があります。

  • 再生 には、再生の URL を使用します。
  • DVR 機能 を持つプレーヤーを使用する

DVR 機能は 86,400 秒に制限されています。

DVR ストリームは、ライブストリームの完了後 7 日間使用できます。

エンドポイントとオペレーション

Live API の主な操作は、ライブ ストリームの作成と管理、ライブストリームからの VOD クリップの生成です。 これらの操作は、次のエンドポイントへの要求を介して実行されます。詳細については、ドキュメントの残りの部分で説明します。

Brightcove Live には、他のアカウント ID とは別の独自のアカウント ID があります。 API オペレーションでは、 account_id は ライブアカウント ID を参照します。

ジョブの作成と管理

これらの操作により、ライブジョブを作成し、その詳細を取得し、停止することができます。 また、広告休憩の即時キューポイントを作成するためのエンドポイントもあります。

ライブジョブの作成

  POST  https://api.bcovlive.io/v1/jobs

このエンドポイントは、 POST 要求を介してライブストリームを作成するために使用されます。 リクエストでは、ライブストリーム自体のプロパティを指定するだけでなく、ライブストリームから生成する VOD クリップを指定することもできます（これは後で エンドポイント からも実行できます）。 リクエスト本文に含めることができるフィールドの詳細は、 API リファレン スに記載されています。

入力プロトコル

ブライトコーブライブは、複数の入力プロトコルをサポートしています。 ジョブを作成するときには、要求本文の protocol フィールドを使用して、使用するジョブを指定します。 サポートされている値は次のとおりです。

  • rtmp (デフォルト)
  • rtp
  • rtp-fec
  • SRT

RTMP プロトコルは、FLV 形式のストリームを配信するためのプロトコルです。 その他のプロトコルは、MPEG2-TS を配信するためのものです。

rt p、r tp-fec 、または srt を使用する場合は、 cidr_whitelist も指定する必要があります（ クラスレスドメイン間ルーティング を参照）。

rtmp を使用する場合は、代わりに入力に ip_whitelist を指定できますが、これは必須ではありません。

RTP+FEC ジョブのリクエストボディの例:

 { 
「live_stream」: true, 
「region」:「us-west-2", 
「reconnect_time」: 300, 
「出力」: [
 {
「label」:「hls720p」, 
「live_stream」: true, 
「height」: 720, 
「video_bitrate」:800, 
「segment_seconds」: 6, 
「keyframe_interval」: 90 
} 
], 
「プロトコル」:「rtp-fec」, 
「cidr_whitelist」: ["127.0.0.1/32"] 
}

ライブ API クイックスタート では、ライブストリームジョブの作成と、それを再生するための Brightcove Player の設定について説明します。

ライブジョブの一覧表示

  GET  https://api.bcovlive.io/v1/jobs

このエンドポイントは、 GET リクエストを介してライブストリームを一覧表示するために使用されます。 エンドポイントは、ページネーション、ソート、および検索フィルタリングをサポートしています。 リクエスト本文に含めることができるフィールドの詳細は、 API リファレン スに記載されています。追加情報については、「 ライブジョブまたは VOD ジョブのリストの取得 」を参照してください。

ライブジョブの詳細を取得

  GET   https://api.bcovlive.io/v1/jobs/:jobId

このエンドポイントを使用すると、ライブストリームに関する詳細情報を取得できます。ライブストリームは、最初にジョブを作成したときにも返されます。 レスポンスフィールドの詳細については、 API リファレン スを参照してください。

アドキューポイントの手動挿入

  POST  https://api.bcovlive.io/v1/jobs/:jobId  /cuepoint

通常、エンコーダーは広告休憩のキューポイントを送信しますが、このエンドポイントにリクエストを送信してすぐに広告休憩を作成することもできます。 詳細については、 API リファレンスを 参照してください。

キューポイントには、「 DD: HH: MM: SS 」という形式の タイムコード が必要です。

ライブジョブの停止

  PUT  https://api.bcovlive.io/v1/jobs/:jobId  /キャンセル

このエンドポイントを使用して、ライブストリームをすぐに停止します。 一度キャンセルすると、ライブ配信を再開することはできません。 詳細については、 API リファレンスを 参照してください。

クリップの作成

ライブストリームからビデオオンデマンドクリップを作成し、Video Cloud アカウントに保存したり、S3 バケットまたは FTP アドレスに送信したりできます。 ライブストリームの作成時にクリップを定義することも、後述するエンドポイントを使用して後でクリップを作成することもできます。 『 クリップを作成する 』マニュアルも参照してください。

VOD クリップの作成

  POST  https://api.bcovlive.io/v1/vods

クリップの開始点と終了点は、ストリームの開始点からのオフセットまたは UNIX タイムスタンプで定義できます。 リクエスト本文フィールドの詳細は、 API リファレン スで確認できます。

ライブ配信のクリップは、イベント終了後 最長7日間 作成できます。

VOD（クリップ）ジョブの一覧を取得する

クリップの VOD ジョブのリストを取得するには、「 ライブまたは VOD ジョブのリストの取得 」および「 API リファレンス 」を参照してください。

SSAI の管理

サーバーサイド広告挿入（ SSAI ）を使用すると、ライブ配信に好きなだけ広告区切りを挿入できます。 また、スレートアセット（VODクリップ）を取り込んで、未使用の広告時間を埋め込むこともできます。

SSAI の設定の詳細については、「 Brightcove Live API を使用したサーバーサイド広告の挿入」および「API リファレンス」を参照してください 。

アカウント広告設定の取得

  GET   https://api.bcovlive.io/v1/ssai/applications/:account_id

このエンドポイントを使用すると、アカウントに設定されているすべての広告設定を取得できます。 応答フィールドの詳細は、 API リファレン スで確認できます。

広告設定の作成

  POST  https://api.bcovlive.io/v1/ssai/application

SSAI の広告を取得する方法を定義する広告設定を作成します。 リクエスト本文フィールドの詳細は、 API リファレンスで確認できます。

広告設定の取得

  GET   https://api.bcovlive.io/v1/ssai/application/:application_id

このエンドポイントを使用して、作成した広告設定の詳細を取得します。 応答フィールドの詳細は、 API リファレン スで確認できます。

広告設定の更新

  PUT   https://api.bcovlive.io/v1/ssai/application/account/:application_id

広告設定の詳細を更新します。 リクエスト本文フィールドの詳細は、 API リファレン スで確認できます。

スレートメディアソースアセットの取得

  GET   https://api.bcovlive.io/v1/ssai/slates/:ACCOUNT_ID

アカウントに定義されているスレートメディアアセットを取得します。 スレートメディアアセットは、広告で埋められない広告の休憩時間を埋めるために使用されます。 応答フィールドの詳細は、 API リファレン スで確認できます。

スレートメディアソース資産の取り込み

  POST  https://api.bcovlive.io/v1/ssai/slates

スレートのメディアアセットを追加して、未記入の広告休憩時間を埋めます。 リクエスト本文フィールドの詳細は、 API リファレン スで確認できます。

スレートメディアソースアセットの削除

  削除   https://api.bcovlive.io/v1/ssai/slates/:SLATE_MSA_ID

スレートメディアアセットを削除します。

静的エントリポイント

静的エントリポイント（SEP）機能を使用すると、エントリポイントの URL と再生 URL を静的に再利用可能にしながら、アクティブ化および非アクティブ化できる、長時間実行されるライブジョブが可能になります。 この機能により、お客様は施設または現場でエンコーダを設定し、ライブチャネルまたはプログラム用の独自のスケジューリングロジックを作成できます。 詳細については、「 静的エントリポイント 」を参照してください。

キャプション

キャプションが h264 入力信号 (user_data パケットで正しくシグナリング) 内にある場合、キャプションは h264 出力に渡されます。

放送エレメンタルライブエンコーダを使用している場合は、SDI（EIA-608/CEA-608）または他のソース（SCTE-20、SCC、文字多重放送、DVB-Sub、補助的、ARIB、TTML、SCTE-27、STL、SRT、SMI）からキャプションを取得し、私たちに送信する h264 ストリームに入れることができます。 他の放送グレードのエンコーダもおそらく同じことを行うことができますが、正式にはテストしていません。

ID3 時限メタデータを挿入

この情報は、「 ID3 時限メタデータの挿入 」に移動されました。

制限事項

  • API を使用して作成されたライブジョブは表示 されず 、 ライブモジュール で使用できません。
  • エンコーダからの初期接続は、Live プレイリストで作成される帯域幅情報を提供します。 初期接続が低い場合、ジョブ構成で出力が高い場合でも、次の処理が行われるまで、プレイリストは同じ情報をプレイリスト上に維持します。
    • エンコーダが再起動される
    • CDN キャッシュもクリアする必要があります。
  • 現在、入力ストリームのフレームレートは 30 FPS に制限されています。 より高いフレームレートの使用に興味がある場合は、サポートにお問い合わせください。
  • デフォルトでは、入力ストリームの解像度は1080pに制限されています。
  • 接続を解除して再接続する場合、ストリーム設定は同じままにする必要があります。 オーディオチャンネル数、解像度、またはコーデック設定を変更すると、予期しない動作が発生します。
  • Video Cloud ビデオのリモートソースに DASH と MP4 を追加できますが、Live では現在 HLS のみをサポートしています。
  • 入力ストリームでは、AAC オーディオのみがサポートされます。
  • アクティブな 待機中の未起動ジョブは、 いつでも最大 5 個まで許可されます。

    コンカレント・ジョブに関するその他の制限事項：

    • チャネル (24 時間 365 日) ジョブの数は、1 リージョンあたり 0 または少ない数 (アカウントの種類によって異なります) に制限されます。
    • 同時に 実行される イベント ジョブの数は、リージョンによって制限されます。通常は 100 です。
    • イベント ジョブの 接続を同時に待機する 数は 5 に制限されています。
    • リージョンごとの SEP ジョブ数は 3 または 10 に制限されています（「 サポートされる AWS リージョン 」を参照）。

    これらの制限は、サポートによってアカウント単位で調整できます。 追加の容量が必要な場合は、アカウントマネージャーにお問い合わせください。

  • ライブジョブの stream _url として返される「RTMP」アドレスは Akamai HD ライブストリームであり、従来の FMS RMTP ストリームではありません。古いバージョンの Internet Explorer ではサポートされていません。
  • ライブストリームは HTTPS 経由で配信されます。Brightcove アカウントで HTTPS が有効になっていない場合、Brightcove プレーヤーはライブストリームの読み込みに失敗します。 お使いのアカウントでオリジン配信の HTTPS サポートが有効になっていない場合は、再生の問題を防ぐために、 ブライトコーブのサポートに連絡 して、オリジン配信の HTTPS サポートを有効にしてください。
  • マルチビットレートの HLS 出力内でトランスミックスされたレンディションを使用する場合、セグメントとキーフレームが入力 GOP 構造に従うように、どの出力でも segment_size と keyframe_interval を避ける必要があります。 そうでない場合、ジョブ作成要求でエラーが返されます。
  • 独自の FTP または S3 オリジンの場所を使用する場合は、オリジンの場所にフォールバックするように CDN を設定する必要があります。 Brightcove Live システムは、ジョブリクエストで提供された CDN のオリジンロケーションを検証しません。
