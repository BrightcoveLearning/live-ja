クイックスタート:ブライトコーブライブ API を使用してライブストリームを作成する

このチュートリアルでは、Live API を使用してライブストリームジョブを作成する手順を説明します。

必要なもの

このチュートリアルを完了するには、次のものが必要です。

  • Live API 用にセットアップされたアカウント
  • Live API の API キー (アカウントの設定時に提供される)
  • Video Cloud アカウント（ライブストリームジョブの作成には必要ありませんが、Brightcove Player での再生の設定には必要ありません）
  • Studio ではなく CMS API を使用して Video Cloud ビデオを作成する場合は、ビデオ/読み取り、およびビデオ/書き込み権限を持つクライアント認証情報が必要です。クライアント認証情報を取得するプロセスに慣れていない場合は、以下の 認証 セクションを参照してください。
  • cURL または 不眠症 や 郵便配達 などのRESTクライアントを介してREST APIリクエストを作成する機能。

    お客様を支援するために、エクササイズに使用できるインソムニアとポストマンの輸入品をご用意しました（ブライトコーブラーニングサービスでは、インソムニアを好み、使用していますが、ポストマンも対応いたします）。

    zipファイルのそれぞれに、あなたと一緒に役立つreadmeページがあります。

    • 不眠症のワークスペースのインポート
    • 郵便配達員の収集と環境のインポート
  • 実際のライブストリームを作成するには、ライブストリーミング用のソフトウェアまたはハードウェアエンコーダが必要です。 以下の手順には、Wirecast を使用してライブストリームをセットアップする手順が含まれています ( 試用版にサインアップ )。 別のエンコーダを使用している場合、プロセスは異なりますが、Live API 応答から必要な情報に関して類似している必要があります。

CMS API の認証情報を取得する

CMS API を使用するには、適切な認証情報が必要です。

ほとんどの場合、資格情報を取得する最も簡単な方法は、Studio Admin API認証セクションを使用する方法です（アカウントに対する管理者権限が必要です）。 詳細は、「 API 認証資格情報の管理 」を参照してください。 この場合、必要な権限は 関係を共有 するためのものです。読み取りと書き込みの両方の権限が必要です。

必要な権限がStudioで利用できない場合、またはOAuth APIから直接取得する場合は、次に示す「 クライアント資格情報の取得 」ドキュメントを選択します。 どちらのオプションを選択しても、正しい操作権限を要求する必要があります。 適切な権限を取得するには、cURL または Postman で以下を使用できます。

 「操作」: [
「ビデオクラウド/ビデオ/読み取り」, 
「ビデオクラウド/ビデオ/作成」, 
「ビデオクラウド/ビデオ/更新」]
  • OAuth：cURLを使用してクライアント認証情報を取得する
  • OAuth：ポストマンを使用してクライアント資格情報を取得する

Video Cloud ビデオを設定する

ライブストリーム情報を取得したので、Video Cloud にリモートビデオを設定し、Brightcove Player で再生できるようにします。

これらのステップでは、 CMS API を使用してビデオを設定しますが、必要に応じて Video Cloud Studio でも実行できます。 以下の代替手順 を参照してください。

InsomniaまたはPostmanを使用している場合は、クライアントの資格情報を使用してOAuth2認証を設定する必要があります。 ヘルプが必要な場合は、次のいずれかのガイドを参照してください。

  • API リクエストにインソムニアを使用する
  • API リクエストに郵便配達員を使用する

  1. まず、以下の POST リクエストを行い、新しいビデオを作成する必要があります。

     https://cms.api.brightcove.com/v1/accounts/  account_id  /videos

  2. リクエストボディには、複数のメタデータを含めることができます（詳細については API リファレンスを 参照してください）。ただし、必要なのは 名前 だけです。

     { 
「name」: " ライブストリームの名前 " 
}

  3. 応答にはすべてのビデオメタデータが含まれていますが、重要なフィールドは ID です。このIDは、リモートアセットを追加する必要があります。

  4. HLS ストリームを追加するには、別の POST 要求を行います。

     https://cms.api.brightcove.com/v1/accounts/  account_id  /videos/  video_id  /assets/hls_manifest

  5. 今回は、リクエストボディは次のようになります。

     { 
「remote_url」: " ライブストリームの再生URL " 
}

プレーヤーにビデオを追加する

ここで、単純なHTMLページを作成し、作成したばかりのビデオでそれにプレーヤーを追加します。

ここでもStudioを使用してプレーヤー公開コードを取得します。必要に応じて、 プレーヤー管理API を使用してこれを行うこともできます。
  1. ライブストリームプレーヤーをテストするための単純な HTML ページを作成します。 あなたが好きな他のコンテンツを含めることもできますし、まったく含めることもありません。
  2. Studio Media モジュールで、前の手順で作成したビデオを選択し、「 公開と埋め込み」>「Web Player 」をクリックします。
  3. [パブリッシュ] ダイアログで、プレーヤーを選択します（Brightcove のデフォルトプレーヤーは問題ありません）。
  4. 下にスクロールし、プレーヤーのサイズを設定してから、コードブロックをクリックして標準プレーヤーコードを選択します。このコードをコピーし、htmlページの本文に貼り付けます。
  5. html ページをリモートまたはローカルの Web サーバーに保存して、参照できるようにします。
  6. Web ブラウザでページを開きます。 私たちは蒸しを開始していないので、あなたはメディアエラーが表示されます：

Studio を使用してビデオを作成する

ここでは、Studio でビデオを作成するための別の手順を示します。

  1. ビデオクラウドスタジオにログインします。
  2. メディアモジュールに移動します。
  3. 左側のオプションで、[ リモートビデオを追加 ] をクリックします。
  4. [ リモートビデオを追加 ] ダイアログで、ビデオ名を追加し、ライブ API レスポンスの playback_url の値をコピーして [URL] フィールドに貼り付けます。 次に、[ レンディションの追加 ] をクリックし、[ 保存 ] をクリックします。
    注：セキュアなストリームのためにHTTPS経由でストリーミングする場合は、playback_urlの http を https に変更するだけです。

Wirecast を使用してイベントをストリーミングする

このセクションでは、ライブストリーミングイベントをサポートするために Telestream Wirecast エンコーダを設定し、使用します。 Telestream Wirecast ソフトウェアは、カメラからの入力をキャプチャし、CDN によって配信できるストリームを生成するデスクトップアプリケーションです。 ライブストリーミングイベントの配信に適したハードウェアやその他のソフトウェアベースのエンコーディングソリューションが用意されています。 Telestream Web サイト から Wirecast ソフトウェアの試用版をダウンロードできます。

注:先に進む前に、Wirecast ソフトウェアをインストールし、カメラをコンピュータに接続しておく必要があります。

ライブイベントの Wirecast を設定するには、次の手順を実行します。

  1. Wirecast を開きます。
  2. + にカーソルを合わせ、 [ デバイスのキャプチャ ] をクリックし、カメラを選択します。
  3. 矢印ボタン () をクリックして、カメラショットがライブショットになります。
  4. [ ストリーム ] ボタン () をクリックします。 出力設定を構成するように求められます。
  5. [ 送信先 ] を [ RTMP サーバー ] に設定します。 [ OK] をクリックします 。
  6. アドレス には、ライブ API から返された stream_url を使用します (ステップ 3)。
  7. ストリームの 場合は、alive を使用します。
  8. （オプション） 追加の出力ストリームを作成するには、[ 追加...] をクリックします。 追加の出力ストリームを作成する場合は、独自のカスタムエンコーディングプロファイルを作成して出力ビットレートを制御することができます。 ストリーム 名が新しいビットレートを反映していることを確認します。
  9. [ OK ] をクリックして、出力設定を保存します。
  10. [ ストリーム ] ボタン () をクリックします。 ライブストリームが開始されます。

結論

これで、ライブストリーミングセッションを作成する準備が整いました。 エンコーダ設定に以下を追加する必要があります。

  • stream_url ：あなたのジョブから stream_url を追加します（ rtmp アドレス）
  • stream_name : ストリーム名は 生きてい なければなりません

ライブ配信を開始すると、前の手順で作成した Brightcove プレーヤーにライブ配信が表示されるはずです。
