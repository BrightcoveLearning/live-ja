ライブ API: ID3 時限メタデータの挿入

このトピックでは、Live API を使用して ID3 時限メタデータをライブストリームに挿入する方法について説明します。

概要

ID3 時限メタデータを使用して、特定の時点でライブストリームを使用してデータを送信します。 進行中のジョブの ID3 時限メタデータを挿入するには、次の POST 要求を実行します。

 https://api.bcovlive.io/v1/jobs/  JOB_ID  /id3tag

サンプルリクエスト本文

 { 
「id3_tag」: {
「名前」:「cp1", 
「値」:「私が送信したいいくつかの情報」, 
「タイムコード」: "15:50:49:16" 
}, 
「id3_tag」: {
「名前」:「cp2", 
「値」:「私が送信したいいくつかのより多くの情報」, 
「タイムコード」:「15:50:16" 
}}} 
}
注：
  • timecode プロパティを使用する場合、ジョブは最新の挿入要求のみを格納します。
  • timecode プロパティを使用する場合、エンコーダは OnFi を介して tc プロパティに格納された SMPTE フォーマット（HH: MM: SS: FF）タイムコードを送信する必要があります。
  • Wirecast や OBS などのソフトウェアエンコーダは、RTMP ストリーム内の OnFi パケット経由のタイムコード送信をサポート していません 。
  • エレメンタルハードウェアエンコーダは、RTMP ストリームの OnFi パケットを介したタイムコードの送信をサポートします。
  • ID3v2 ステートのタグ 名は 最大 4 文字です。
  • 値は 最大 256 KB です。
  • タイムコードは オプションです。省略すると、挿入は即時に行われます。

サンプル応答

 { 
「id」:「JOB_ID」, 
「ID3_タグ」: {
「タグ名」:「BCOV」, 
「タグ値」:「私の値」
} 
}

入力に SMPTE タイムコードが存在しますか?

すべてのエンコーダが SMPTE タイムコードを送信するわけではありません。 入力にそれらが存在するかどうかを確認するには、 #VOD-timing: TimeCode= 15:18:36:01、wallclock=1549047672441、tsvideo=25400、tssaudio=-1 のようなタグの VOD チャンクリストを調べることです。 ジョブ情報を取得するときに、vod チャンクリストの URL が output_media_files [0] .playback_url_vod で使用できるはずです。

ライブプレイリストを分析してタイムコードを取得する

Brightcove Live ジョブを作成すると、デフォルトのマスター再生リストが返されます。 たとえば、次のようになります。

 https://bcovlive-a.akamaihd.net/ade7a7b3aaea4a4d8107fb8bb262381e/eu-central-1/52725330351/playlist.m3u8

VODマスタープレイリストが必要です。このプレイリストには、URL拡張子の最後の部分に「_vod」を playlist_vod.m3u8 として追加する必要があります。

 https://bcovlive-a.akamaihd.net/ade7a7b3aaea4a4d8107fb8bb262381e/eu-central-1/52725330351/playlist_vod.m3u8

プレイリストは複数のレンディション/プロファイルで構成され、プレイリストを「カール」するとレンディションを見ることができます。 例:

  
 
curl-i "https://bcovlive-a.akamaihd.net/ade7a7b3aaea4a4d8107fb8bb262381e/eu-central-1/52725330351/playlist_vod.m3u8"
