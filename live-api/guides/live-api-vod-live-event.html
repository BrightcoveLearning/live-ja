ライブAPI：ライブイベントからのVOD

このトピックでは、イベントの完了後にライブストリーミングイベントからビデオオンデマンド (VOD) を作成する方法について説明します。
Live to VOD は暗号化されたストリームでは使用できず、Live ジョブイベントの作成時にエラーが返されます。

はじめに

短いライブストリーミングイベントの場合は、イベントの完了後に Video Cloud アカウントでイベントの録画をビデオ オンデマンド（VOD）として使用できます。 これは、 ライブジョブを作成 するときに VOD の設定を追加することで簡単に行うことができます。

Fast Live to VOD では、クリップをセグメント精度で迅速に公開できます。また、顧客に対して完全に透過的なバックエンドプロセスによって、クリップの境界がクリーンアップされ、再公開されます。

その結果、目的のクリップ境界で自動的に再公開されるクリップを使用して、市場投入までの時間が短縮されます。

重要 :

  • ライブストリーミングイベントが完了したら、エンコーダを停止しますが、 ジョブ をキャンセル しないでください 。 VOD クリップが作成されるまで待ってから、ジョブをキャンセルします。そうしないと、正常に作成されません。
  • ライブジョブの reconnect_tim e が期限切れになるまで、VOD の作成は開始されません。
  • ライブイベントの期間が 24 時間を超える場合は、最後の 24 時間のみが VOD にキャプチャされます。

高速VODオプション

高速VODオプションを使用すると、ライブイベントのVODバージョンをより迅速に取得できます。 Fast Live to VOD は、 セグメント精度の高い ライブレンディションを（トランスマルチックスを使用して）パブリッシュします。

高速VOD の出力 フィールドは次のとおりです。

  • モード （高速VODに必要）-高速VODの場合、値は インスタント になります
  • playlist_label （オプション）-インスタントモードの VOD にのみ適用されます。 （ライブジョブから）VOD 出力を作成するために使用するプレイリストを示します。 ライブジョブの作成時にカスタムプレイリストが定義されている場合にのみ必要です。

実装

ビデオクラウド に送信

新しいライブジョブを作成するときに videocrowd オブジェクトを使用する方法は 2 つあります。

  1. ビデオ クラウドを ジョブレベルのフィールドに 含めると、Vide o Cloud アカウントにビデオ が作成され、ライブストリームをリモートアセットとして受け取ります。

「region」:「us-west-2",

「live_dvr_sliding_window_duration」: 30, 
 「videocloud」: {
「video」: {
「name」:「test_live_stream」
}} 
} ...

    この場合、何も 取り 込まれないため、 ビデオクラウド オブジェクト内に取り込みオブジェクトはありません。

S3 バケットの認証情報がまだ設定されていない場合は、 API を使用して 新しい認証情報を追加できます。

ビデオメタデータについては、CMS API の「 ビデオの作成 」操作にリストされた任意のフィールドを含めることができます。

取り込みプロファイルは、VOD 用に作成されるレンディションを定義します。 valid-ingest-profile-name には、アカウントのカスタムまたは標準の取り込みプロファイルの名前を置き換えることができます。 どの標準プロファイルがあるかは、アカウントで動的配信が有効になっているかによって異なります。 「 動的配信およびコンテキスト対応エンコーディングの標準取り込みプロファイル 」または「 レガシートランスコーディングの標準取り込みプロファイル 」を参照してください。

プロファイル フィールドはオプションであることに注意してください。 プロファイルを指定しない場合、アカウントのデフォルトのプロファイルが使用されます。

制限事項

  • VOD が作成される前にライブジョブを 停止 (キャンセル) した場合、そのジョブは作成されません。 代わりに、エンコーダを停止して VOD の作成を開始し、その完了を待ってからジョブを停止します。
  • スタティックエントリポイント ストリームの 場合のみ 、ライブ to VOD は完了時にストリームをアーカイブするように設定できません。

    代替:

    • クリッピング を使用して、ライブイベント全体をVODとしてアーカイブします。 クリッピングのウィンドウは、イベントの発生後、または SEP の次回アクティブ化 (どちらか短い方) までの標準期間で使用できます。
