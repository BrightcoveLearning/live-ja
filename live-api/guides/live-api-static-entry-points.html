ライブ API: 静的エントリポイント

このトピックでは、静的エントリポイントと、Live API を使用して静的エントリポイントを実装する方法について説明します。

はじめに

静的エントリポイント（SEP）機能を使用すると、エントリポイントの URL と再生 URL を静的に再利用可能にしながら、アクティブ化および非アクティブ化できる、長時間実行されるライブジョブが可能になります。 この機能により、お客様は施設または現場でエンコーダを設定し、ライブチャネルまたはプログラム用の独自のスケジューリングロジックを作成できます。

DVR および SEP

DVR 機能は、スタティックエントリポイントで使用できますが、SEP がアクティブ になっている間だけ使用できます。

静的エントリポイントの作成

標準の Create Job 操作が使用されますが、静的エントリポイントを追加するには、ジョブ要求にプロパティを追加する必要があります。

 「静的」: 真

このジョブからの応答 RTMP URL は静的であり、次のセクションで説明する操作を使用してジョブをアクティブ/非アクティブにすることで再利用できます。 このドキュメントの最後に記載されている制限事項を読み、理解してください。

最大待機時間

SEP ジョブがアクティブ化された後、エンコーダが起動されていない場合、しばらくすると自動的に非アクティブになります。 ルールは次のとおりです。

  • event_length の値が 30 分を超える場合、ジョブはデフォルトで 30 分後に自動的に終了します。待機時間を変更する場合は、サポートに連絡する必要があります。 自動非アクティブ化を無効にすることはできません。
  • event_length の値が reconnect_time より小さい場合、ジョブは event_length に設定された時間で終了します。

SEP ライブストリームのアクティブ化

静的エントリポイントのライブストリームを有効にするには、Live API に次のリクエストを送信します。

静的エントリポイントに取り込むには、API の アクティブ化 リクエストを行う必要があります。
アクティブな SEP ストリーム
メソッド PUT
[端点] 
 https://api.bcovlive.io/v1/jobs/  {jobId} / アクティブ化

リクエスト本文は必要ありません。

SEP ライブストリームの非アクティブ化

注：ストリームを無効にする前に、エンコーダを必ず停止してください。

静的エントリポイントのライブストリームを無効にするには、Live API に次のリクエストを送信します。

アクティブな SEP ストリーム
メソッド PUT
[端点] 
 https://api.bcovlive.io/v1/jobs/  {jobId} / 非アクティブ化
静的なエントリポイントを非アクティブにすると、ストリームの最後の数秒は再生可能なままであることに注意してください。

リクエスト本文は必要ありません。

SEP ジョブのキャンセル

静的エントリポイントジョブを永続的に停止するには、エンドポイント /jobs/ {job_id} /cancel に PU T リクエストを送信することで、ライブジョブと同様にジョブをキャンセル できます。 詳細については、 API リファレンスを 参照してください。

制限事項

  • ライブ to VOD は、完了時にストリームをアーカイブするように設定できません。

    代替:

    • クリッピング を使用して、ライブイベント全体をVODとしてアーカイブします。 クリッピングのウィンドウは、イベントの発生後、または SEP の次回アクティブ化 (どちらか短い方) までの標準期間で使用できます。
  • ジョブ設定を変更できません。 現在、SEP を最初に作成した設定に関連付けられています。
  • ストリームをアクティブ化するには、エントリポイントが使用可能になるまでに最大 60 秒かかることがあります。

    ベストプラクティス：

    ライブプラットフォームからジョブの状態変更に関する通知を実装して、SEP ジョブが実際に準備完了したときのリアルタイムの更新を取得します。

    代替:

    1. エンコーダを起動する前に、少なくとも 60 秒前にストリームをアクティブにします。
    2. エンコーダにリトライ（Wirecast、Elemental など）が組み込まれている場合、ストリームは即座に接続でき（エラーがある可能性もある）、エントリポイントが利用可能になるとすぐにエンコーダが再接続されます。
  • reconnect_time は、エンコーダの切断時の SEP ジョブにも適用されます。 再接続時間に達すると、SEP ジョブは非アクティブ状態に戻ります。

    ベストプラクティス：

    ジョブの完了時には常に API を介してストリームを非アクティブにし、切断時に待機する許容可能な時間に再接続時間を設定します (この待機期間中に入力時間コストが発生します)。

  • 各アカウントには、利用可能な SEP ジョブが 3 つだけ割り当てられます。 上限に達した場合、新しい SEP ジョブを作成する前に SEP ジョブの 1 つ をキャンセル する必要があります。

    SEP ジョブの最大数に到達し、別のジョブを作成しようとすると、次のエラーが返されます。

     { 
「error」:「アカウント用に作成できる静的エントリポイントのジョブ数に達しました」
}

注記

非アクティブ化後の SEP ジョブのクリッピング
ユーザーは、ストリームの次回のアクティブ化まで、または 7 日間のいずれか短い方のクリップを作成できます。 ストリームが再びアクティブになると、以前の録画セッションはサーバーから削除され、クリップできなくなります。
SEP が使用されたときに stream_start_time フィールドのタイミングを教えてください。
SEP がアクティブ化されると、開始時刻と見なされます。
