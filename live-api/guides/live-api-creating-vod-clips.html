ライブ API: VOD クリップの作成

このトピックでは、ライブ配信からビデオオンデマンド（VOD）クリップを作成する方法について説明します。

概要

クリップは、ライブストリームから抽出されたビデオです。 これらは S3 バケット、FTP サイト、または Video Cloud アカウントに送信できます。 クリップはMP4ビデオとして作成され、すべてのケースで宛先に送信されます。 Video Cloud の場合、MP4 は取り込みシステムによってトランスコードされ、ビデオ用に作成されるレンディションの種類は、使用する取り込みプロファイルによって異なります。

クリップの定義は、 /vods エンドポイントを使用して作成されます。

クリップは、いくつかの方法で作成できます。

  • ライブストリームイベントの SMPTE タイムコードで stream_start_timecode または stream_end_ tim ecode が定義されている場合、エンコーダがタイムコード情報を送信する必要があることに注意してください 。
  • ライブストリームイベント全体の開始 時間（stream _ start _time）を基準に定義された開始時間（stream_start_tim e）
  • 開始時刻および/または 終了 時刻 をエポック（Unix）時間で定義した場合（秒単位）
  • 期間 あり
  • VOD API は、 暗号化された ジョブ では使用できません 。

注記

  1. 従来の取り込みプロファイルがライブクリッピングをサポートしなくなりました。 ライブ配信からビデオクリップを作成するには、アカウントで動的配信が有効になっていることを確認し、動的配信取り込みプロファイルを使用します。
  2. クリップをできるだけ早く利用できるようにするために、セグメント精度のクリップが最初に作成され、利用可能になったらすぐにフレーム精度のクリップに置き換えられます。
  3. デュレーション を指定した場合、作成されるクリップは次のようになります。
    • ジョブがアクティブでまだ存続している場合:(要求時間-期間) ～ (要求時間)
    • ジョブが終了した場合:( 終了時 -期間) から ( 終了時 )
  4. 開始時刻と 終了 時刻 の両方を指定した場合:
    • ジョブがアクティブでまだライブの場合：エポック時間ウィンドウが完全に created_at とリクエスト時間内に収まる限り、クリップが作成されます。
    • ジョブが終了した場合：エポックタイムウィンドウが created_at および finished_at 内に完全に収まる限り、クリップが作成されます。
  5. SSAI を使用したライブ配信のクリップには広告は含まれません。
  6. クリップは、イベント後7日間まで作成できます。 SEP では、次のアクティベーションまで、または 7 日 (どちらか短い方) まで作成できます。
  7. VOD API は、ストリーム内に存在するコンテンツ以外のコンテンツを追加しません。 300 秒のライブストリームで 350 を指定した場合、出力は 300 秒になります。
  8. クリッピングを実行するために、DVR 対応のライブストリームを使用する必要はありません。ライブストリームはブロードキャスト時に保存され、イベントの終了後すぐに 7 日間使用できます。
  9. Brightcove Live クリッピングは、最高解像度の出力と同じ解像度のクリップのみを生成します。 ソース入力解像度と一致しません（最高解像度の出力と同じでない限り）。

クリップは複数の宛先に送信することもできます。

  • ビデオクラウド アカウント
  • FTP サーバー
  • S3 バケット

クリップを指定する場合、 ビデオの作成と Video Cloud でのクリップの取り込みについて詳しく説明するために、出力先に URL または videocrowd オブジェクトが含まれている 必要があります 。

注： ライブストリームの実行中にクリップを作成できます 。 これを行うには、クリップの開始時刻と終了時刻をエポック時間、またはライブストリームの 開始 時刻を基準に定義する必要があります。

資格情報

クリップの送信先がアクセスするために認証情報が必要な場合は、Live API の認証情報オペレーションを使用して認証情報を作成できます。 詳細については、「 Live API の認証情報の管理 」を参照してください。

ライブアカウントに関連付けられている Video Cloud アカウントにクリップを送信する場合は、クリッピングジョブに資格情報を含め ないでください 。
