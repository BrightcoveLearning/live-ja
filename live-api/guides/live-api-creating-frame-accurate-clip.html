ライブ API: フレーム精度クリップの作成

このチュートリアルでは、Live API を使用してライブジョブを作成し、そこからフレーム精度の高いクリップを作成する手順を説明します。

概要

このチュートリアルでは、 ブライトコーブの ライブでライブストリーミングジョブを作成し、そこからフレーム精度の高いクリップを作成する方法を API レベルで説明します。

フレーム精度クリッピングでは、エンコーダが SMPTE タイムコードを送信する必要があることに注意してください。

このチュートリアルでは、 curl を使用してAPIリクエストを作成しますが、 Insomnia 、 Postman 、または他のRESTクライアントで簡単に作成できます。

ライブジョブの作成

まず、ライブジョブを作成します。 ライブAPIには API-KEY が必要です。 アクセスできない場合や、アクセスの取得に興味がある場合は、アカウントマネージャーにお問い合わせください。

  1. すでにブライトコーブがあり、API-KEY を持っていると仮定します。 bcov-live-api-key と呼びましょう。

  2. 次の curl コマンドをコピーしてテキストエディタに貼り付けます。

エンコーダを設定する

ここに示す手順は、このチュートリアルの手順をテストするために使用された Elemental Live ボックス エンコーダを想定しています。 エンコーダが異なる場合は、設定が似ているはずです。

この場合の最も重要な設定は次のとおりです。

  • タイムコードソースを「システムクロック」として設定
  • 「OnFi タイムコード周波数」を 1 に設定します。
  • ビデオ内の「タイムコード挿入」をチェックする
  • 強く推奨：フレームレート = ソースに従う

フレーム精度の高いクリップを作成する

このチュートリアルでは、クリップを Video Cloud ライブラリに送信することを前提としています。 クリップを S3 バケットまたは他の場所に送信する場合、プロセスは同じです。API リクエストのリクエスト本文にはわずかな違いがあります。

資格情報

Live アカウントで Video Cloud の認証情報を作成していない場合は、以下の手順に従います。 これは 1 回限りの操作です。

Video Cloud へのビデオの動的取り込みのクライアント認証情報がまだない場合は、作成する必要があります。 Studioでこれを行う方法については、「 API認証資格情報の管理 」を参照してください。

  1. 以下の curl コマンドをテキストエディタにコピーします。
     curl-X POST\ https://api.bcovlive.io/v1/credentials\-H 'コンテンツタイプ:アプリケーション/json'\-H 'x-api-key: '\-d' {「クレデンシャル_default_for_type」: true,「クレデンシャル_ラベル」:「Video_Cloud_uploads」,「クレデンシャル_private」:「」,「クレデンシャル_type」:

  2. 置換:

    • をライブ API キーで
    • クライアントシークレットで
    • をクライアントIDで変更する
  3. 編集したコマンドをコピーしてターミナルまたはコマンドラインアプリに貼り付け、実行します。
  4. レスポンスは次のようになります。
     { 
「credential_id」:「1e0180330b724dfbcaf6b28b6c5d517", 
「user_id」:「c2691d4d039040be96c190a949d754a7", 
「credential_label」:「Video_Cloud_Uploads」
}

クリップを作成する

Video Cloud の認証情報を取得したら、クリップを作成する準備が整います。

  1. 以下の curl リクエストをコピーし、テキストエディタに貼り付けます。
     curl-X POST\ 
 https://api.bcovlive.io/v1/vods\ 
-H 'Content-Type: application/json'\ 
-H 'x-api-key: '\ 
-d' {
「live_job_id」: "」, 
「outputs」: [{
「label」:「SMPTE TC 17:33:11:12 から 17:34:00:00」, 
「stream_start_タイムコード」:「」, 
「stream_end_timecode」: 
「」,「ビデオクラウド」: {"取り込み」: {"キャプチャイメージ」: true}, "ビデオ」: {"名前」: "テストフレーム精度クリップ"}, 
「資格情報」:「」
}] 
} '

  2. 置換:

    • をブライトコーブのライブ API キーに置き換えます。 たとえば、 Feg-this-a-fake-API-key-FgjaSD12HJHSZ
    • を、ライブジョブの作成リクエストに対する応答のジョブ ID に置き換えます。 この例では、この値 は3b6871bf2f344acaa6b397d09b476018 になります。あなたの値は異なります。
    • であり、ライブストリームに意味がある（存在している）値 でなければなりません。 この例では、使用しています： 17:33:11:12 と 17:34:00:00
    • を Video Cloud 認証情報のラベルに置き換えます。 まだ作成しておらず、上記の手順を使用して作成した場合は、 Video_Cloud_Uplo ads になります。
  3. 編集したコードをターミナルまたはコマンドラインアプリにコピーして貼り付け、実行します。
  4. レスポンスは次のようになります。
     { 
「vod_jobs」: [
 {
「jvod_id」:「0b14a4ba326d4d08f15053ca2a403b7
",「ラベル」:「SMPTEタイムコードによるトリム」
} 
], 
「live_job_id」:「3b6871bf2f344acaa6b397d09b476018" 
}

私たちは、クリップをダウンロードし、 Adobe Premiere を使用して最初と最後のフレームをチェックした実験の精度をテストしました。タイムコードがオーバーレイされているので、簡単な視覚的なチェックは、入力ポイントと出力ポイントの精度を確認するのに十分です。
