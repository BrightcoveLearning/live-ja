ライブ API: 通知

このトピックでは、Live API ジョブの通知を設定する方法を示します。

はじめに

Brightcove Live システムは、さまざまなイベントの通知を送信します。 通知のリスナーを設定して、さらにアクションをトリガーできます。 リスナーは、使用するサーバー側の言語で記述することができ、受信したPOSTリクエストを探し、JSONリクエスト本文を解析し、必要なアクションを実行します。 以下に、通知をリクエストする方法を示します。

通知のリクエスト

ジョブ作成 リクエストの出力に 1 つ以上の 通知 フィールドを含めることで、通知をリクエストできます。

通知の値は、通知 先オブジェクトの配列です。 URL: "http://httpbin.org/post"、または {"url」: " http://httpbin.org/post「、" 資格情報」: "reference_to_your_credentials"} のオプションを持つオブジェクトを使用できます。 出力の状態が変更されると、指定した宛先に通知が送信されます。 このパラメータを VOD 出力に適用すると、通知はその VOD ジョブにのみ関連し、ライブジョブには関連しません。

ご注意ください。 イベントタイプ（ first_segment_uploaded、state_ change など）ごとに1つの通知URLのみが許可されます。

再試行戦略

通知の送信要求が失敗した場合、デフォルトの再試行方法は、試行間隔を指数関数的に遅延して 50 回再試行することです。

 max_retry_times = 50 
 delay_delta_s = 5 
 next_retry = now_s + retry_count * delay_delta_s

イベント

以下は、あなたが聴くことができるジョブライフサイクルのイベントです。 イベントスコープはジョブ単位であるため、ライブジョブのイベントライフサイクルは、同じライブストリームに基づく VOD クリッピングジョブのライフサイクルとは別になります。

  • state_chang ed-変更されたライブジョブの状態。詳細については、以下の表を参照してください。
  • first_segment_upload ed-ライブストリームの最初のセグメントがオリジンにアップロードされます。
  • output_finished-ライブイベントが終了 し、VOD 処理によって少なくとも 1 つのオーディオと 1 つのビデオレンディション、または 1 つのプログレッシブ MP4 レンディションが作成されました。

Live モジュールでの通知の設定

Live モジュールで作成されたライブイベントの通知を設定するには、次の操作を行う必要があります。

  1. 前のセクションで示した PHP アプリケーションのような POST 要求を受信できるハンドラーアプリケーションを作成します。
  2. パブリック URL でアプリをホストします。
  3. Live モジュールで Live ジョブを作成する場合は、[ 詳細オプション ] を展開します。
  4. [ ストリームステータス通知を有効にする] オプションをオンに し、ハンドラーアプリの URL を入力します。
