ライブ API によるライブ DRM

このトピックでは、Live API を使用したライブストリームの DRM の追加について説明します。

はじめに

DRM は、ジョブの作成時にリクエスト本文に DRM オブジェクトを含めることでライブジョブに追加できます。 DRM オブジェクトには、次の 2 つのプロパティがあります。

  • モード を配列に設定するか、「すべて」、「フェアプレイ」、「プレイレディ」、「ワイドバイン」、「ワイドバイン:ダッシュ」、「ワイドバイン:hls」のいずれかまたは複数に設定します。
  • t@@ oken_expires_in に 43200（デフォルト）から 31622400（トークンの有効期限が切れるまでの秒数）までの数値を設定します。

モード を all に設定すると、使用可能なすべての DRM タイプが出力に適用されます。 デフォルトでは、DRM は次のように出力に適用されます。

  • フェアプレイ:HLS 出力に適用
  • プレイレディ:DASH 出力に適用
  • ワイドバイン：HLS出力に適用

ワイドバインは、HLSの代わりにDASH出力に適用することができます。 DASH で Widevine を作成するには、デフォルトでアカウントレベルの上書きが必要です。 この上書きをアカウントに適用する場合は、サポートにお問い合わせください。 次のセクションでは、アカウントのデフォルトに関係なく、特定のジョブに対して widevine: hls または widevine: dash を指定する方法について説明します。

マイクロソフトとのサービス契約で定義されている PlayReady には同時実行性の制限があります。 同時実行性の管理については、 この Microsoft ドキュメント を参照してください。 動画の同時実行制限を超えると、最大数を超える視聴者の再生に失敗します。
