JobstartStop

JobStartStopエンドポイントを使用すると、既存のライブジョブのアクティブ化と非アクティブ化をスケジュールできます。指定された時刻に、当社のシステムは、お客様に代わってジョブをアクティブ化および非アクティブ化するために必要なAPI呼び出しを行います。

必要条件

  • アカウントのライブ API キー。
  • ライブアカウントでは、スケジューラのオーバーライドを有効にする必要があります。
  • 「静的」: true の既存のライブジョブ。

JobStartStop ワークフローを作成する

2021-02-02T 14:00:00 + 00:00 に始まる2時間のイベントをスケジュールするとしましょう。

タイムスタンプは、秒単位のUNIXタイムスタンプとしてUTC形式である必要があります。https://www.unixtimestamp.com/で変換できます。

ここで説明するアクティベーション時間の制限は、スケジュールされたアクティベーションにも適用されるため、希望する開始時刻の少なくとも 60 秒前までの時間をスケジュールすることをお勧めします。

この例では、開始予定時刻の5分前を選択します。

始める：2021-02-02T13：55：00 + 00：00 = 1612274100停止：2021-02-02T 16:00:00 + 00:00 = 1612281600

ワークフローを作成するには、次のリクエストを行います

POST https://api.bcovlive.io/v1/scheduler/jobstartstopヘッダー：コンテンツタイプ:アプリケーション/json X-API-KEY: あなたの API キー本文: { "job_id": "your Job ID", "activation": { "time_utc": 1612274100 } ,「非アクティブ化」: { "time_utc": 1612281600 } ,「説明」:「私の火曜日の放送」}

次のような応答を受け取るはずです

{ "account_id": "your Account ID", "activation": { "state": "pending", "time_utc": 1612274100 } ,「非アクティブ化」: { "state": "pending", "time_utc": 1612281600 } ,「説明」:「私の火曜日の放送」,「job_id」:「あなたのジョブID」,「タイプ」:「jobstartstop」,「workflow_id」:「d44cf29f4f184757ac0995beb8a0097e」}

JobStartStop ワークフローを更新する

JobStartStopワークフローのアクティブ化時間と非アクティブ化時間を更新できます。これらは「状態」:「保留中」の場合にのみ更新できます。

前の例に従って、スケジュールより30分早くイベントを停止します。

PUT https://api.bcovlive.io/v1/scheduler/jobstartstop/workflow_idヘッダー：コンテンツタイプ:アプリケーション/json X-API-KEY: API キー本体: { "deactivation": { "time_utc": 1612279800 } }

応答で更新されたワークフローを受け取ります。

{ "account_id": "your Account ID", "activation": { "state": "pending", "time_utc": 1612274100 } ,「非アクティブ化」: { "state": "pending", "time_utc": 1612279800 } ,「説明」:「私の火曜日の放送」,「job_id」:「あなたのジョブID」,「タイプ」:「jobstartstop」,「workflow_id」:「d44cf29f4f184757ac0995beb8a0097e」}

JobStartStop ワークフローをキャンセルする

DELETEリクエストを使用して、アクティベーションと非アクティブ化をキャンセルできます。アクティブ化後にこれを実行して、非アクティブ化をキャンセルすることができます。

https://api.bcovlive.io/v1/scheduler/jobstartstop/workflow_id を削除:X-API-KEY: あなたの API キー

応答で更新されたワークフローを受け取ります。

{ "account_id": "your Account ID", "activation": { "state": "cancelled", "time_utc": 1612274100 } ,「非アクティブ化」: { "state": "cancelled", "time_utc": 1612279800 } ,「説明」:「私の火曜日の放送」,「job_id」:「あなたのジョブID」,「タイプ」:「jobstartstop」,「workflow_id」:「d44cf29f4f184757ac0995beb8a0097e」}

通知

オプションで、ワークフローの作成時に通知を構成できます。POSTするサービスの URLを提供する必要があります。スケジューラーがアクティブ化および非アクティブ化の呼び出しを行うと、通知が送信されます。アクティベーションまたは非アクティブ化の n秒前に通知を受信するようにを設定することもできます。詳細については、「通知ガイド」を参照してください。

POST https://api.bcovlive.io/v1/scheduler/jobstartstopヘッダー：コンテンツタイプ:アプリケーション/json X-API-KEY: あなたの API キー本文: { "job_id": "your Job ID", "activation": { "time_utc": 1612274100, "notification": 600 } ,「非アクティブ化」: { "time_utc": 1612281600, "notification": 600 } ,「説明」:「私の火曜日の放送」、「notification_url」: "http://example.com/live/jobstartstop/callbacks"}
