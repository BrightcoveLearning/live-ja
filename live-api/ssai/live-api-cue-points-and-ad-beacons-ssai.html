ライブ API: SSAI を使用したキューポイントとアドビーコン

このトピックでは、ライブ配信ジョブにブライトコーブのサーバーサイド広告挿入（SSAI）を使用する際のキューポイントと広告ビーコンを管理する方法について説明します。

概要

サーバーサイド広告挿入 （SSAI） を使用すると、ライブストリーミングイベント中に指定した時間に広告を表示できます。 一般的な情報については、 Live API: サーバーサイド広告挿入 (SSAI) ドキュメントを参照してください。

SSAI を使用するには、アカウントで SSAI を有効にしておく必要があります。 詳細については、アカウントマネージャーにお問い合わせください。

キューポイント

広告ブレークは、キューポイントによってトリガーされます。キューポイントは次の2つの方法で指定できます。

  • エンコーダによってブライトコーブに送信
  • ライブ API を使用して作成された即時キューポイント

エンコーダから

ブライトコーブのライブ配信システムは、エンコーダから送信されたキューポイントを AMF 形式で解釈できます。

 AMFDataList 
 [0]: OnCuePoint 
 [1]: {Obj []: 
 時間:1.9889, //アドブレイク
 名の 1 番目のビデオフレームの最初の PTS に対するこのパケットの PTS との差:「scte35」, 
 タイプ:「event」, 
 ad_server_data:「ywjjzgVMZ2G=」,//任意で導入された Brightcove. これは、パラメータのbase64でエンコードされたjsonマップです。例えば {'key'：'value'} 
 パラメータ：{Obj []：
 タイプ：「avail_in"、
 期間：12.0 
} 
}

注意事項:

  • avail_in タイプのキューポイントのみが現在サポートされています。
  • RTMP および TS 入力では、SCTE-35 キューポイントがサポートされています。
Elemental エンコーダのキューポイント設定のサンプルスクリーンショットについては、 以下の付録 を参照してください。

手動キューポイント挿入

POST リクエストを送信することで、 Live API を使用して即時キューポイントを作成できます。

メソッド POST
URL https://api.bcovlive.io/v1/jobs/ Job_id /キューポイント
ヘッダー X-APIキー： あなたのAPIキー

以下を指定するリクエストボディを含めます。

フィールド タイプ 説明
期間 整数 休憩時間（秒単位）。

挿入するキューポイントの持続時間は、ジョブ内の セグメントの長さの 2 倍以上に する必要があります。 期間の例 を参照してください。
タイムコード SMPTE フォーマット オプション: SMPTE 形式のタイムコード ( HH: MM: SS: FF (FF = フレーム) で、変数のセット (キー/値のペア) を ADServer に渡すタイミングを指定します。

省略すると、キューポイントがすぐに挿入されます。

timecode プロパティを使用する場合、エンコーダは onFi を介して t c プロパティに格納された SMPTE フォーマット（ HH: MM: SS: FF ）タイムコードを送信する必要があります。 タイムコードはライブストリームの開始時からのものです。
server オブジェクト オプション: 渡すキーと値のペアは、使用している広告サーバーによって異なります。 詳細については、広告サーバーのドキュメントと「 広告マクロを使用したターゲティング広告 」セクションを参照してください。

期間の例

挿入するキューポイントの持続時間は、ジョブ内の セグメントの長さの 2 倍以上に する必要があります。

例えば、「segment_seconds」が 4 のジョブに 10 秒のキューポイント を挿入すると正常に動作します。 ただし、「segment_seconds」が 6 のジョブに同じキューポイントを挿入すると、以下のエラーが発生します。

 「error」:「パラメータ期間はジョブの (2 * ターゲット期間) 
 以上である必要があります」
 

サンプルリクエスト本文

 { 
「期間」: 30, 
「タイムコード」: "15:50:49:16「, 
「ad_server_data」: {
「adbreakid」: 12312 
「breaktheme」:「フィットネス」
} 
}

注記

  1. Wirecast や OBS などのソフトウェアエンコーダは、RTMP ストリーム内の OnFi パケット経由のタイムコード送信をサポートしていません。
  2. エレメンタルハードウェアエンコーダは、RTMP ストリームの OnFi パケットを介したタイムコードの送信をサポートします。

SCTE-35 広告変数

S ociety of Cable Telecomision Engineers (SCTE) は、ライブ配信の動的広告挿入の標準を定義している。 次の表は、SCTE-35 広告の設定変数をまとめたものです。

SCTE-35 広告パラメータは、現在、RTMP 入力ソースとトランスポートストリーム（TS）入力ソースの両方でサポートされています。リアルタイム転送プロトコル（RTP）は、フォワードエラー訂正（FEC）の有無にかかわらず、リアルタイム転送プロトコル（RTP）を使用します。 RTP には FEC を使用することを強くお勧めします。

これらの広告パラメータは、リアルタイムメッセージングプロトコル（RTMP）を使用した入力ストリームでは使用できません。
SCTE-35 広告設定変数
変数
説明
eventID
SCTE35 メッセージとともに渡される一意のイベント ID。
プログラムID
SCTE35 メッセージとともに渡された一意のプログラム ID。
AvailNum
広告に使用可能な特定のスプライス時間の ID。SCTE35 メッセージを介して送信されます。
BreakDuration
SCTE35 メッセージとともに渡された、プログラムの 90 kHz クロックのティックを基準とした広告休憩の継続時間。
SpliceTime
SCTE35 メッセージとともに渡された、プログラムの 90 kHz クロックのティックの観点から、広告休憩のスプライス時間。

ヘッダー X-APIキー： あなたのAPIキー

 { 
「ビーコンセットID」：「ビーコンセットID」、
「削除済み」：真
