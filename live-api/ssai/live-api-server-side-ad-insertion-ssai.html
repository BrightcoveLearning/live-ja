ブライトコーブライブ API: サーバーサイド広告挿入 (SSAI)

このトピックでは、ライブ配信ジョブにサーバーサイド広告挿入（SSAI）を設定する方法について説明します。

概要

SSAI では、ライブストリーミングイベント中に指定した時間に広告を表示できます。 主な機能は次のとおりです。

  • エンコーダから送信されたキューポイントを使用して広告を挿入するか、 Live API を使用してインスタントキューポイントを作成します。
  • 「スレート」アセットを取り込み、未使用の広告時間を埋める
  • サーバー側でライブ配信に張り付く広告で広告ブロッカーを避ける
SSAI を使用するには、アカウントで SSAI を有効にしておく必要があります。 詳細については、アカウントマネージャーにお問い合わせください。

サーバーサイド広告でライブ配信を作成するには、次の手順に従います。

  1. ライブ API に関する一般情報の確認
  2. ライブ広告の設定を作成します 。 Video Cloud Studio でもこれを行うことができます。 Live API を使用した広告設定の管理について詳しくは、以下のセクションをご覧ください。
  3. スレートアセットを作成します 。 UIを使用する場合は、 Studioでスレートを管理 できます。
  4. オプション: キューポイントと広告ビーコンを挿入します 。
  5. これで、Live API を使用してライブストリームを作成する 準備が整いました。 Studioを使用する場合は、 サーバーサイド広告をライブモジュールで実装 できます。

カスタムヘッダー、広告設定変数、DFP および広告マクロを使用した詳細については、このドキュメントの残りの部分を参照してください。

一般情報

以下の情報は、SSAI リクエストを持つすべての Live API に関するものです。

ベース URL

SSAI を使用したライブ API のベース URL は次のとおりです。

 https://api.bcovlive.io/v1/ssai

認証

リクエストの認証には、API キーを持つヘッダーが必要です。

DFPに関する注意事項

DFP から広告を取得する場合は、問題を防ぐために留意すべき点をいくつかご紹介します。

広告タグ

Live の広告タグを作成するときは、適切なガイドラインに従い、 /live を含めてください。 詳細については、DFP ドキュメント「 マスタービデオタグを手動で作成する 」を参照してください。

同時実行性

同時実行性が高いことが予想される場合は、DFP アカウントチームにお問い合わせいただくことをお勧めします。

単一/複数広告の応答

シングルリードレスポン スと マルチアドレスポンスの パラメータは、現在 SSAI では使用されていません。

Live SSAI は 1 つの広告サーバー呼び出しのみを行い、応答にすべての広告を含めることを期待します。ただし、広告ごとに 5 つのリダイレクトに制限がある広告ラッパーに従うことを除きます。 次の広告応答形式が受け入れられ、次のように解析されます。

  • VAST-単一の応答または単一の応答での広告のポッド
  • DFP広告ルール-事前定義、ミッドロール、ポストロール定義広告を含む、レスポンスで利用可能なすべての広告を集計します。
  • Smart XML-事前定義、ミッドロール、ポストロール定義広告を含む、すべての利用可能な広告をレスポンスに集約します。

広告リクエストのカスタムヘッダー

SSAI プラットフォームは、バックエンドプラットフォームで使用される広告呼び出しとすべてのビーコンでカスタムヘッダーを渡すことができます。 VideoPlaza などの一部の広告サーバーでは、カスタムヘッダーが必要です。

カスタムヘッダーは、 ad_configuration_head ers オブジェクトのキーと値のペアのセットとして指定されます。このオブジェクトは application_ad_ configuration の一部です（「 広告設定を作成する 」セクションを参照してください）。

注記

  • 標準ヘッダーは、次のようにデフォルトで処理されます。
    • X 転送対象
    • X デバイスユーザエージェント
  • ヘッダー値には、 広告設定変数
  • ヘッダー値は静的文字列でもかまいません
ヘッダーは、広告ラッパーやインプレッション/四分位数を含むすべてのダウンストリームリクエストに渡されるため、注意して使用する必要があります。

広告マクロを使用した広告のターゲティング

広告設定を作成すると 、通常、広告タグは次のようになります。

 https://ad-provider-host.com/path/to/ad-handler?ip={{client.ipaddress} & 
 num= {{{random.int32}} &ses= {{session.sess
 ion_id}} &IDFA= {{deviceid}} & サイトセクション= {{{sitesection}} &breakid= {{breakid}} &breaktheme= {{{breaktheme}}}}}

二重中括弧内の項目は、広告マクロとも呼ばれる変数です。Brightcove Live は、広告リクエストを送信するときに、値が存在する場合は、値に置き換えます。

有用な変数は、広告サーバーによって異なります。

広告マクロの値は、次の方法で指定できます。

  • ヘッダー情報の使用
  • URL の追加
  • URL の動的な追加
  • 手動キューポイントの使用

ヘッダー情報の使用

ヘッダー情報は、上記の [ 広告設定変数 ] セクションで詳述されており、どのリクエストでも使用できます。 広告設定で、必要な変数を適切なキー名で指定するだけです。

 playlist.m3u8 ？ デバイスID=123456&サイトセクション=サービス  "

広告設定変数

広告設定変数は、ビーコンとも呼ばれ、広告設定を管理するためのリクエストで使用できます。 詳細については、 ライブ API: SSAI を使用したキューポイントと広告ビーコン ドキュメントを参照してください。

手動キューポイントの使用

特定の広告休憩の値は、手動キューポイント挿入リクエストで送信できます。 詳細については、「 ライブ API: キューポイントと広告ビーコン 」ドキュメントを参照してください。

既知の問題

  • SSAI では、ライブストリーミングビデオにオーディオトラックが必要です。
  • 返された VAST の広告 ID が同じ場合、広告 URL が動的変数を使用して一意の URL にしていても、サーバーは広告コンテンツを要求しません。 これは、広告に DFP を使用している場合は 適用されません 。
  • DFP では、同じ広告 ID を使用できますが、別のクリエイティブ ID が必要です。つまり、MediaFile を簡単にスワップすることはできません。

  • 広告休憩期間が、広告URLの最大期間よりも短い場合（min_ad_duration=0&max_ad_duration=30000）、広告休憩時間よりも長い広告が返された場合、広告は再生されません。

  • ブライトコーブライブ SSAI では、VPAI D 広告またはクリック可能広告はサポートされていません。

  • 広告休憩の残り時間がストリームのセグメント秒よりも短く、スレートが表示されている場合、そのセグメント期間中スレートが表示され、実際の広告休憩時間は予想以上に長くなります。

  • Google のシステムで初めて広告が表示されたときは、トランスコードされて配信準備が整うまで広告は再生されません。
  • ライブ SSAI 用の VMAP は現在サポートされていません。
