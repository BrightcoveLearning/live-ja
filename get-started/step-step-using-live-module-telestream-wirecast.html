    ステップバイステップ:ライブモジュールと三ストリームWirecastの使用

    このクイックスタートの目的は、Video Cloud Live モジュールと Tlestream Wirecast を使用してライブイベントを作成する方法を示すことです。
     

    ライブモジュールで使用できるすべての機能については、「ライブモジュールを使用したライブイベントの作成と管理」を参照してください。高品質で安定したライブストリーミングエクスペリエンスを提供するために役立つベストプラクティスおよび推奨事項については「ライブモジュールガイドラインおよびベストプラクティス」を参照してください。ブライトコーブのライブストリーミング動画配信オプションの一覧については「ライブストリーミング動画を配信する」を参照してください。

    注：OBS エンコーダを使用するこのマニュアルのバージョンについては、ステップバイステップを参照してください。Open Broadcaster ソフトウェア (OBS) でライブモジュールを使用する
    注：ライブストリーミングのサポートをご利用いただけるのは、Video Cloud ProおよびVideo Cloud Enterpriseをご利用のパブリッシャーのみです。

    このクイックスタートを完了すると、次の事項ができるようになります。

    • Video Cloud Liveモジュールを使用してライブイベントを作成する
    • ライブイベント用にエンコーダーを設定する
    • イベントをトリミングしてVideo CloudアカウントにVODアセットとして保存する

    このクイックスタートは、Telstream Wirecast を使用してさまざまなデバイスにライブイベントをブロードキャストするのに役立ちます。三次元サイトで無料トライアルをご利用いただけます。Liveモジュールでは他のエンコーダーも使用できます。

    対象者

    Video Cloud Liveモジュールを使用してライブストリーミングイベントを放送する必要があるVideo Cloudパブリッシャー

    前提条件

    • ライブモジュールにアクセスできるビデオクラウドプロまたはエンタープライズアカウント
    • イベントを放送するためのカメラ
    • 独自のエンコードソフトウェアが必要です。このクイックスタートでは、無料でダウンロードできる Telestream Wirecast を使用します ( サポートされているエンコーダの一覧を参照 )。

    制約事項

    • お客様のエンコーダーでは、RTMPストリームを出力する必要があります。
    • ライブ放送では20～30秒の遅延が発生します。
    • ストリーミングは、イベント開始から 30 分以内に開始する必要があります (既定のアカウント設定)
    • Internet Explorerを使用している場合、バージョン10以降が必須です。

    ライブイベントをブロードキャストする手順

    Liveモジュールを使用してライブイベントをストリーミングするには、次の手順を行う必要があります。

    • Video Cloud Live モジュールを使用して新しいイベントを作成する
    • エンコーダを設定する
    • ライブストリームを公開する
    • ライブイベントからクリップを作成する

    新しいライブイベントを作成する

    1. ブラウザーウィンドウで Video Cloud Studio に移動します。
    2. Video Cloudログイン資格情報を入力し、[サインイン]をクリックします。
    3. お客様の現在のVideo Cloudアカウント名がページの右上隅に表示されます。複数のアカウントをお持ちの場合は、アカウントセレクタードロップダウンをクリックして、ライブアセットを保存したいアカウントを選択してください。Liveが選択したアカウントでまだ有効でない場合、Liveモジュールがトライアルモードで開きます。
    4. ナビゲーションヘッダーで[ライブ]をクリックします。
    5. [ イベントを作成 ] ボタンをクリックして、新しいイベントを作成します。
    6. イベント名を入力します。
    7. イベントのタグをいくつか入力します。
    8. アカウントに必須のカスタムフィールドが設定されている場合は、カスタムフィールドに値が割り当てられていることを確認してください。
    9. ドロップダウンリストから [ ライブインジェスト] プロファイルを選択します。ライブ取り込みプロファイルは、作成されるライブレンディションの数と品質を制御します。
    10. 完了したら、[イベントをビデオアセットに変換 ] を選択します。これにより、イベントがビデオとして Media モジュールに保存されます。
    11. [ 詳細オプション ] リンクをクリックして、詳細オプションを展開します。
    12. [ 再接続時間 ] を 1 分に設定します。
    13. お住まいの場所に最も近い地域を選択します。
    14. [ イベントを作成 ] をクリックします。ライブモジュールは、ストリームエンドポイント (RTMP URL) とストリーム名を返します。これらの値はエンコーダー設定で使用されます。

    エンコーダの設定

    このセクションでは、ライブストリーミングイベントをサポートするようにTelestream Wirecastエンコーダーを設定し、使用します。Telestream Wirecastソフトウェアは、カメラから入力を取り込み、CDNによって配信できるストリームを生成するデスクトップアプリケーションです。ハードウェアベースおよび他のソフトウェアベースのエンコーディングソリューションには、ご利用のライブストリーミングイベントの配信に、より適したものが存在する可能性があります。Wirecastソフトウェアの体験版を TelestreamのWebサイトからダウンロードすることができます。

    注：先に進む前に、Wirecastソフトウェアをインストールし、カメラをコンピュータに接続する必要があります
    注：この例では、Blackmagic Design UltraStudio ミニレコーダーを使用してラップトップに接続されたデジタル一眼レフを使用します。
    注：Wirecast には、ストリームのローカルコピーを記録する機能があります。これを選択すると、イベントのブロードキャスト中にネットワークの問題が発生した場合に備えて、ストリームのローカルコピーが作成され、そこからクリップを作成できます。ローカルファイルを記録する方法については、 Wirecast のマニュアルを参照してください。

    この例では、Blackmagic Design UltraStudio ミニレコーダーを使用してラップトップに接続されたデジタル一眼レフを使用します。

    1. Wirecastを開きます。
    2. [ + ] をクリックし、[ ビデオキャプチャ ] を選択します。
    3. リストからカメラを選択します。[追加]をクリックします。
    4. 選択したビデオソースがプレビューペインに表示されていることを確認します。
    5. 矢印ボタン ( ) をクリックして、カメラをライブショットにします。
    6. ストリームボタン（）をクリックします。出力設定を入力するよう表示があります。
    7. 出力先を RTMP サーバーに設定します。[OK]をクリックします。
    8. アドレスには、ライブモジュールが提供するストリーム URL を使用します。
    9. ストリームの場合は、生きているを使用してください。
    10. ストリームボタン（）をクリックします。ライブストリームが始まるはずです。
    注：ストリーム処理の所要時間は約1分間です。

    ライブイベントの公開

    ライブイベントの埋め込みコードを生成するには、次の手順に従います。

    1. ライブモジュールに戻ります。
    2. イベントの [Control Room] ページが表示されない場合は、[ ライブ ] タブをクリックしてライブイベントのリストを表示します。イベント名をクリックして、「コントロールルーム」ページを開きます。
    3. ライブストリームを視聴できるかを確認します。エンコーダの起動後、ライブストリームが表示されるまでに1分かかる場合があります。
    4. [ パブリッシュして埋め込む ] をクリックします。
    5. プレーヤーを選択し、プレーヤーの URL をクリックしてライブストリームを表示します。
    6. ライブストリームを終了したら、ライブモジュールの [ Stop Stream ] ボタンをクリックします。また、Wirecast に戻り、ストリームボタンをクリックして、エンコーダのストリーミングを停止します。
      注：ライブストリームの完了後、エンコーダーを閉じる/停止することが重要です。エンコーダーとブライトコーブとの間の接続を開いたままにしておくと、データが送信されていなくても、予期しない接続料が発生する場合があります。

    クリップの作成

    ライブモジュールは、完了したイベントからビデオクリップを作成する機能を提供します。ライブイベントの進行中にクリップを作成することもできます。クリップを作成したい理由はいくつかあります。

    • ソーシャルメディアに投稿できる短いティーザークリップを作成するには
    • 投稿する動画の長さを制限するには（ほとんどのプラットフォームでは動画の長さが制限されています）
    • ソーシャルメディアで短い動画のパフォーマンスが向上することがわかりました

    クリップは、イベント終了から7日以内にしか作成できません。その後、[ クリップを作成 ] ボタンは無効になります。イベントが 24 時間よりも長い場合は、最後の 24 時間だけがクリップ可能です。

    注：クリッピングインタフェースの使用方法の詳細については、ライブモジュールを使用したビデオクリップの作成を参照してください。

    クリップを作成するには、次の手順に従います。

    1. ライブモジュールに戻ります。
    2. [ 完了 ] タブをクリックします。
    3. イベントを検索し、イベント名をクリックして Control Room を開きます。
    4. [ クリップを作成 ] をクリックします。クリッピングエディタが開きます。
    5. フォーカスタイムラインのクリッピングハンドル ( ) をクリックしてドラッグし、クリップの開始点と終了点を設定します。
    6. [ クリップをプレビュー ] をクリックして、クリップをプレビューします。必要に応じて、始点と終点を調整できます。
    7. [ クリップを保存 ] をクリックします。
    8. クリップ名とその他のメタデータ値を入力します。アカウントの必須フィールドも表示され、値を割り当てる必要があります。
    9. [ クリップを作成 ] をクリックします。取り込みプロセスが開始され、クリップが Video Cloud アカウントに保存されます。
    10. クリップはクリップエディタの下に表示されます。[ ソーシャルに公開 ] をクリックして、Brightcove Social を使用していずれかのソーシャル配信先にクリップを公開します。Social モジュールを使用したビデオの公開については、「ソーシャルメディアプラットフォームへのビデオの公開」を参照してください。
    11. コントロールルームに戻る:ブレッドクラムが完成しました。すべてのクリップは、イベントのコントロールルームにも表示されます。  
      注：[クリップの作成]ボタンを数回クリックして、複数の放送クリップを作成することができます。これにより、各クリップを個別のVODアセットとしてメディアライブラリーに保存し、各放送用に何回でもトリミングして保存することができます。

    ライブストリームの問題をトラブルシューティングする

    お客様のライブイベントが機能するよう、さらにサポートが必要な場合は、弊社までお問い合わせください。なるべく早くご返信できるようにするため、サポートチームが問題解決に必要な情報のリストを以下に示します。

    • ストリームに見受けられる特定の兆候。たとえば、一部が再生できない、スムーズに再生できないなど
    • 問題のストリームが過去に正しく機能していたか
    • エンコーダーで使用しているエントリーポイントURL
    • 使用しているエンコーディングソフトウェアおよびハードウェア
    • ライブイベントを公開したプレーヤーのURL
    • Video Cloud Studio内のライブアセットの動画ID
    • エンコーダーから公開ポイントホストへのルートトラッキングの結果
