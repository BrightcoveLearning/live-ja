ステップバイステップ：オープンブロードキャスタソフトウェア（OBS）でのライブモジュールの使用

このクイックスタートの目的は、Video Cloud Live モジュールとOpen Broadcaster Software (OBS) を使用してライブイベントをブロードキャストする方法を示すことです。
 

Live モジュールで使用できるすべての機能については、「Live モジュール を使用したライブイベントの作成と管理 」を参照してください。 高品質で安定したライブストリーミング体験を実現するためのベストプラクティスと推奨事項については、「 ライブモジュールのガイドラインとベストプラクティス 」を参照してください。 ライブストリーミング動画を配信するための Brightcove で利用可能なすべてのオプションのリストについては、「 ライブストリームの配信 」を参照してください。

注：Telestream Wirecast エンコーダを使用するこのドキュメントのバージョンについては、「 ステップバイステップ:Telestream Wirecast でのライブモジュールの使用 」を参照してください。
注:ライブストリーミングのサポートは、Video Cloud Pro および Enterprise のパブリッシャーでのみ利用できます。

このクイックスタートを完了すると、次のことができるようになります。

  • Video Cloud Live モジュールを使用してライブイベントを作成する
  • ライブイベント用にエンコーダを設定する
  • イベントをトリミングして、Video Cloud アカウントに VOD アセットとして保存する

このクイックスタートは、Open Broadcaster Software（OBS）Studioを使用して、ライブイベントをさまざまなデバイスにブロードキャストするのに役立ちます。 OBSは、ビデオ録画とライブストリーミングのためのオープンソースソフトウェアであり、 ここでダウンロード することができます。 他のエンコーダは、Live モジュールとともに使用することもできます。

観客

Video Cloud Live モジュールを使用してライブストリーミングイベントをブロードキャストする必要がある Video Cloud パブリッシャー。

前提条件

  • Live モジュールにアクセスできる Video Cloud Pro アカウントまたはエンタープライズアカウント
  • イベントをブロードキャストするカメラ
  • 独自のエンコーディングソフトウェアが必要です。このクイックスタートでは、無料でダウンロードできる OBS Studio を使用しています（ サポートされているエンコーダ の一覧を参照）

制限事項

  • エンコーダは RTMP ストリームまたは SRT ストリームを出力する必要があります。
  • ライブ放送は、20-30 秒の遅延が発生します。
  • ストリーミングは、イベント開始から 30 分以内に開始する必要があります (デフォルトレベルのアカウント設定)。
  • Internet Explorer を使用するユーザーには、バージョン 10 以降が必要です。

ライブイベントをブロードキャストする手順

Live モジュールを使用してライブイベントをストリーミングするには、次のタスクを実行します。

  • Video Cloud Live モジュールを使用して新しいイベントを作成する
  • エンコーダを設定する
  • ライブ配信を公開する
  • ライブイベントからクリップを作成する

新しいライブイベントの作成

  1. ブラウザウィンドウで、 Video Cloud Studio に移動します。
  2. Video Cloud のログイン認証情報を入力し、[ サインイン] をクリックします。
  3. 現在の Video Cloud アカウント名がページの右上隅に表示されます。 複数のアカウントがある場合は、アカウントセレクタのドロップダウンをクリックし、ライブアセットを保存するアカウントを選択します。 Live が有効になっていないアカウントを選択すると、Live モジュールは試用モードで開きます。
  4. ナビゲーションヘッダーの [ ライブ ] をクリックします。
  5. [ Create Event ] ボタンをクリックして、新しいイベントを作成します。
  6. イベント名 を入力します。
  7. イベントの タグ を入力します。
  8. アカウントが必須のカスタムフィールドで構成されている場合は、カスタムフィールドに値が割り当てられていることを確認してください。
  9. ドロップダウンリストから ライブ取り込みプロファイル を選択します。 ライブ取り込みプロファイルは、作成されるライブレンディションの数と品質を制御します。
  10. 完了したら、イベントをビデオアセットに変換 を選択します。 これにより、イベントがメディアモジュールにビデオとして保存されます。
  11. 詳細オプション リンクをクリックして、詳細オプションを展開します。
  12. [ 再接続時間 ] を 1 分に設定します。
  13. お 住まいの地域に最も近い地域 を選択します。
    OBS Encoder は SRT プロトコルをサポートしており、[ 詳細オプション ] メニューの [ 入力形式 ] セクションでこのオプションを選択できます。
  14. [ ストリーミングの開始 ] をクリックします。 ライブモジュールは、 ストリームエンドポイント（RTMP URL） と ストリーム名 を返します。 これらの値は、エンコーダ設定で使用されます。
    ライブイベントにSRTプロトコルを選択した場合は、ストリーム設定サービスで「カスタム」を選択し、サーバーボックスに srt：// パスを入力することで、OBSで利用可能になります。

エンコーダの設定

このセクションでは、ライブストリーミングイベントをサポートするために OBS Studio エンコーダを設定および使用します。 OBS Studio ソフトウェアは、カメラからの入力をキャプチャし、CDN によって配信できるストリームを生成するデスクトップアプリケーションです。 ライブストリーミングイベントの配信に適したハードウェアやその他のソフトウェアベースのエンコーディングソリューションが用意されています。 OBS Studioソフトウェア はこちら からダウンロードできます。

注：先に進む前に、OBSソフトウェアをインストールし、カメラをコンピュータに接続しておく必要があります。
注：この例では、Blackmagic Design UltraStudioミニレコーダーを使用してラップトップに接続されたデジタル一眼レフを使用しています。
注：OBSは、ストリームのローカルコピーを記録する機能を提供しています。 これを選択すると、イベントのブロードキャスト中にネットワークの問題が発生した場合に備えて、クリップの作成元となるストリームのローカルコピーが作成されます。 ローカルファイルを記録する方法については、 OBSのマニュアル を参照してください。
  1. OBS スタジオを開きます。
  2. [ ソース ] セクションで、[ + ] をクリックして新しいソースを追加します。 リストから ソース を選択します。 このクイックスタートは、Blackmagic Design UltraStudioミニレコーダーを使用してラップトップに接続されたデジタル一眼レフを使用しているため、 Blackmagic Device が選択されています。
  3. 新しいソースの名前を入力し、[ OK ] をクリックします。
  4. デバイス リストからカメラを選択します。 プロパティダイアログにプレビューが表示されます。 [ OK] をクリックします 。
  5. [コントロール] セクションで、[ 設定 ] をクリックします。
  6. 設定ダイアログの左側にある「 ストリーム 」をクリックします。
  7. [ ストリームの種類 ] で、[ カスタムストリーミングサーバー ] を選択します。
  8. URL には、ライブモジュールが提供する ストリーム URL を使用します。
  9. Stream キー の場合は、 alive を使用します。
  10. [ OK ] をクリックして設定を保存します。
  11. [コントロール] セクションで、[ ストリーミングの開始 ] をクリックします。 ライブストリームが開始されます。
注:ストリームの処理には約 1 分かかります。

ライブイベントの公開

ライブイベントの埋め込みコードを生成するには、次の手順に従います。

  1. Live モジュールに戻ります。
  2. イベントの [Control Room] ページが表示されない場合は、[ Live ] タブをクリックしてライブイベントのリストを表示します。 イベント名をクリックして、「コントロールルーム」ページを開きます。
  3. ライブ配信が表示されることを確認します。 エンコーダの起動後、ライブストリームが表示されるまで 1 分かかる場合があります。
  4. [ パブリッシュと埋め込み ] をクリックします。
  5. プレーヤーを選択し、プレーヤー の URL をクリックしてライブストリームを表示します。
  6. ライブストリームの処理が終了したら、ライブモジュールの「 ストリームを停止 」ボタンをクリックします。 また、OBS Studio に戻り、ストリームボタンをクリックして、エンコーダのストリーミングを停止します。
    注：ライブ配信が完了したら、エンコーダを閉じたり停止したりすることが重要です。 データが送信されていない場合でも、エンコーダとブライトコーブ間の接続を開いたままにしておくと、予期しない接続料金が発生する可能性があります。

クリップの作成

Live モジュールは、完了したイベントからビデオクリップを作成する機能を提供します。 ライブイベントの進行中にクリップを作成することもできます。 クリップを作成する理由はいくつかあります。

  • ソーシャルメディアに投稿できる短いティーザークリップを作成するには
  • 投稿する動画の長さを制限するには（ほとんどのプラットフォームは動画の長さを制限します）
  • あなたは、短い動画がソーシャルメディアでより良いパフォーマンスを発見しました

クリップは、イベント終了後7日以内に作成することができます。 その後、[ クリップを作成 ] ボタンは無効になります。 イベントが 24 時間を超える場合は、過去 24 時間のみが切り取られます。

注：クリッピングインタフェースの使用方法の詳細については、「 Live モジュールを使用したビデオクリップの作成 」を参照してください。

クリップを作成するには、次の手順に従います。

  1. Live モジュールに戻ります。
  2. [ 完了 ] タブをクリックします。
  3. イベントを探し、イベント名をクリックしてコントロールルームを開きます。
  4. [ クリップを作成 ] をクリックします。 クリッピングエディタが開きます。
  5. フォーカスタイライン上のクリッピングハンドル () をクリックしてドラッグし、クリップの開始点と終了点を設定します。
  6. クリップを プレビューするには、[クリップ をプレビュー] をクリックします。 開始点と終了点は必要に応じて調整できます。
  7. [ クリップを保存 ] をクリックします。
  8. クリップ名と その他のメタデータ値を入力します。 アカウントの必須フィールドも表示され、値を割り当てる必要があります。
  9. [ クリップを作成 ] をクリックします。 取り込みプロセスが開始され、クリップが Video Cloud アカウントに保存されます。
  10. クリップがクリップエディタの下に表示されます。 [ Publish to Social ] をクリックして、Brightcove Social を使用してクリップをソーシャルサイトの 1 つにパブリッシュします。 Social モジュールを使用したビデオの公開については、「 ソーシャルメディアプラットフォームへのビデオの公開 」を参照してください。
  11. [ コントロールルームに戻る:完了 ] ブレッドクラムをクリックします。 すべてのクリップは、イベントのコントロールルームにも表示されます。
    注:[Create Clip] ボタンを複数回クリックすると、ブロードキャストの複数のクリップを作成できます。 これにより、各放送で必要な回数だけトリミングして保存し、クリップを個別の VOD アセットとしてメディアライブラリに保存できます。

ライブ配信に関する問題のトラブルシューティング

ライブイベントを動作させるためにさらにサポートが必要な場合は、 当社までご連絡ください 。 可能な限り最速の応答が得られるように、問題を解決するために必要なサポートのリストを以下に示します。

  • ストリームに発生している特定の症状。 例えば、それはまったく再生されないのですか、それは吃音またはフリーズしますか？
  • このストリームが過去に正しく機能したかどうか
  • エンコーダで使用しているエントリポイント URL
  • エンコーディングソフトウェアとハードウェアは、
  • ライブイベントを公開したプレイヤーの URL
  • Video Cloud Studio でのライブアセットのビデオ ID
  • エンコーダからパブリッシングポイントホストへのトレースルートの結果
